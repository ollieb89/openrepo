Create `workspace/occc/src/components/memory/ConflictPanel.tsx`:

**Props interface:**
```typescript
interface ConflictPanelProps {
  flaggedItem: MemoryItem;
  conflictItem: MemoryItem;
  similarityScore: number;
  onEdit: (id: string, content: string) => Promise<void>;
  onDelete: (id: string) => Promise<void>;
  onDismiss: () => void;
  onClose: () => void;
  onAdvanceNext: () => void;  // called after resolution
}
```

**Layout — slide-in side panel from right:**
- Fixed position, right-aligned, full height, width ~50vw (or max-w-2xl)
- Semi-transparent backdrop overlay (click to close)
- Smooth slide-in transition using `translate-x-full` → `translate-x-0` with `transition-transform duration-300`
- Header: "Conflict Resolution" title, similarity score badge (e.g., "92% similar"), close X button

**Side-by-side content area:**
- Two columns, each showing one memory's content
- Column headers: "Memory A" / "Memory B" with truncated IDs
- Implement lightweight word-diff highlighting:
  - Split both contents by whitespace into word arrays
  - Two-pass comparison: find common words vs. additions/deletions
  - Mark differing words with `<mark>` spans — additions in green background, deletions in red background with strikethrough
  - Use a simple LCS (longest common subsequence) approach or sequential diff — no library needed, ~30 lines of code
  - Dark mode support for highlight colors

**State management:**
- `editing: boolean` — controls edit mode
- `editContent: string` — editable content (initialized from flaggedItem.content)
- `resolving: boolean` — loading state during API calls

**Three action buttons at bottom:**
1. **Edit** — toggles `editing` state. When editing:
   - Left column content becomes a `<textarea>` pre-filled with flaggedItem.content
   - Action buttons change to "Save" (calls `onEdit(flaggedItem.id, editContent)`) and "Cancel"
   - After save succeeds: set `resolving = false`, call `onAdvanceNext()`
2. **Delete** — shows inline confirmation ("Delete this memory?") then calls `onDelete(flaggedItem.id)`. After success, calls `onAdvanceNext()`.
3. **Dismiss** — calls `onDismiss()` to remove the flag from healthFlags map, then calls `onAdvanceNext()`

**Auto-advance logic:**
- `onAdvanceNext` is provided by MemoryPanel — it finds the next unresolved flag in the ordered flag list and updates conflictPanel state to show it
- If no more flags, closes the panel entirely
- Brief opacity transition (150ms fade) between items for visual continuity

Use Tailwind classes consistent with existing components. Match dark mode patterns from MemoryRow.tsx.
