  <action>
The event bridge is a Python daemon thread that starts when any process imports the `openclaw` package. Neither the gateway (Node.js) nor the dashboard (Next.js) imports Python, so the bridge socket is absent unless a Python process is running.

**Step 1 — Check if event bridge is already running:**

```bash
curl -sf http://localhost:6987/occc/api/health | python3 -m json.tool 2>/dev/null | grep -A2 "event_bridge"
```

If output shows `"status": "healthy"` — bridge is already running. Skip to Step 3 (confirm).

**Step 2 — Start the event bridge in the background:**

```bash
nohup openclaw-monitor tail --project pumplai > /tmp/openclaw-monitor.log 2>&1 &
echo "Monitor PID: $!"
```

Wait 3 seconds for the daemon thread to initialize:

```bash
sleep 3
```

**Step 3 — Confirm event bridge healthy:**

```bash
curl -sf http://localhost:6987/occc/api/health
```

Parse the response and confirm `event_bridge.status` is `"healthy"`. The full health response should look like:

```json
{
  "gateway": { "status": "healthy" },
  "memory": { "status": "healthy" },
  "event_bridge": { "status": "healthy" }
}
```

If `event_bridge.status` is still `"unhealthy"` after 3 seconds:
- Check the monitor log: `cat /tmp/openclaw-monitor.log`
- Try waiting longer: `sleep 5 && curl -sf http://localhost:6987/occc/api/health`
- If still unhealthy after 10 seconds total: STOP and report the exact error from `event_bridge.error` field in the health response and from `/tmp/openclaw-monitor.log`. Do not proceed to criterion execution.

**Step 4 — Confirm SSE endpoint no longer returns engine_offline:**
