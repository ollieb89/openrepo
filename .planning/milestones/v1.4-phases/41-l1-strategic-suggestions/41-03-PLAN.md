  - `runError: string | null`

- **Run analysis handler:**
  ```typescript
  async function handleRunAnalysis() {
    if (!projectId || isRunning) return;
    setIsRunning(true);
    setRunError(null);
    try {
      await fetch(`/api/suggestions?project=${projectId}`, { method: 'POST' });
      await mutate();  // Refresh data after run
    } catch (e) {
      setRunError('Analysis failed. Check that memU is running.');
    } finally {
      setIsRunning(false);
    }
  }
  ```

- **Accept/reject handlers** — passed down to SuggestionCard:
  ```typescript
  async function handleAccept(suggestion: Suggestion) {
    const res = await fetch(`/api/suggestions/${suggestion.id}/action`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ action: 'accept', project: projectId, diff_text: suggestion.diff_text }),
    });
    if (res.ok) {
      await mutate();  // Pitfall 6: refresh immediately after action
    }
  }

  async function handleReject(suggestion: Suggestion, reason: string) {
    await fetch(`/api/suggestions/${suggestion.id}/action`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ action: 'reject', project: projectId, rejection_reason: reason }),
    });
    await mutate();
  }
  ```

- **Render layout:**

  ```
  [Header: "SOUL Suggestions"]
  [Last run: {data.last_run formatted} | Run analysis button (spinner when isRunning)]
  [Tab bar: Pending (count) | Dismissed]
  [Tab content: list of SuggestionCard or DismissedTab]
  ```

**Accept-as-is only (locked CONTEXT.md decision):** NO textarea for editing diff_text before accepting. The diff shown is exactly what gets sent to the API.

**Accept flow:**
- Set `accepting = true`
- Call `onAccept(suggestion)` — handles API call
- On success: set `accepted = true`, show inline: `<span className="text-green-600 text-sm">Applied to soul-override.md</span>`
- On error from parent: set `error = "Accept failed"`

**Reject flow:**
- First click "Reject" → set `rejecting = true` → show text input with placeholder "Why reject? (optional)" + "Dismiss" and "Cancel" buttons
- "Dismiss" click → call `onReject(suggestion, rejectReason)` → card re-renders (parent mutates, card disappears from pending)
- "Cancel" → set `rejecting = false`, clear rejectReason

Use `<pre>` tag with `className="bg-gray-900 text-gray-100 p-3 rounded text-xs overflow-auto font-mono"` for diff display.

Evidence examples: small gray list below the diff, format: `T-042: Ignored --output-path flag, wrote to default location`

This is a `'use client'` component. Add state and effect at the top of the component:
```typescript
const [pendingCount, setPendingCount] = useState(0);
// Read project from localStorage to avoid ProjectContext dependency in Sidebar
const [projectId, setProjectId] = useState<string | null>(null);

useEffect(() => {
  if (typeof window !== 'undefined') {
    setProjectId(localStorage.getItem('occc-project'));
  }
}, []);

useEffect(() => {
  if (!projectId) return;
  const fetchCount = async () => {
    try {
      const res = await fetch(`/api/suggestions?project=${projectId}`);
      const data = await res.json();
      const count = (data.suggestions || []).filter((s: {status: string}) => s.status === 'pending').length;
      setPendingCount(count);
    } catch {
      // Silently ignore — badge is non-critical
    }
  };
  fetchCount();
  const interval = setInterval(fetchCount, 30000);  // 30s poll
  return () => clearInterval(interval);
}, [projectId]);
```
