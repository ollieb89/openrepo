## Integration Gaps (Completed Phases)

### Gap 1: entrypoint.sh hardcodes `main` branch (CFG-06, severity: medium)
- **File:** `docker/l3-specialist/entrypoint.sh:39`
- **Issue:** L3 container creates staging branch with `git checkout -b "${STAGING_BRANCH}" main` â€” bypasses the `_detect_default_branch()` helper that Phase 16 correctly wired on the L2 side
- **Fix:** Pass `DEFAULT_BRANCH` env var from `spawn_l3_specialist()`, use `${DEFAULT_BRANCH:-main}` in entrypoint

### Gap 2: `get_state_path`/`get_snapshot_dir` not in `orchestration/__init__.py` (CFG-03, severity: low)
- **File:** `orchestration/__init__.py`
- **Issue:** Phase 11 CFG-03 functions importable via `from orchestration.project_config import ...` (all current callers use this) but absent from package `__all__`
- **Fix:** Add `get_state_path`, `get_snapshot_dir`, `ProjectNotFoundError` to `__init__.py` exports

### Gap 3: `soul_renderer` has no runtime trigger (CFG-04/05, severity: low-medium)
- **File:** `orchestration/soul_renderer.py`
- **Issue:** `render_soul()` and `write_soul()` exist and work correctly but are not called by any runtime path â€” requires manual CLI invocation for new projects
- **Fix:** Call `write_soul(project_id)` in `initialize_workspace()` or document as explicit operator step

### Gap 4: `geriai/project.json` wrong ID (MPR-03, severity: low)
- **File:** `projects/geriai/project.json`
- **Issue:** Contains `"id": "pumplai"` â€” copy-paste error from template. Would cause wrong project identity in SOUL rendering and config resolution
- **Fix:** Update ID and all project-specific values for geriai

## Broken E2E Flows

### Flow 1: L3 Container Staging Branch Creation
- **Steps complete:** Project env injection, state file wiring, container naming, label assignment
- **Breaks at:** `entrypoint.sh:39` â€” hardcoded `main` as branch base instead of using dynamic detection
- **Impact:** Repos with non-`main` default branch would create orphan staging branches inside L3 containers

### Flow 2: Multi-Project SOUL.md Provisioning
- **Steps complete:** Template exists, override mechanism works, renderer produces correct output
- **Breaks at:** No runtime call site triggers `render_soul()` during agent initialization
- **Impact:** New projects don't auto-receive a SOUL.md â€” operator must manually run CLI
