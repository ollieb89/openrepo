    **propose.py** — Refactor main() into a session-based flow:

    1. Keep existing argument parsing, outcome resolution, project resolution, registry/config loading, clarification, and initial generate->lint->score->classify pipeline UNCHANGED.

    2. After building the initial ProposalSet and rendering output, check `_is_interactive()`. If not interactive OR `--json` flag, return 0 (preserve existing behavior).

    3. If interactive, create a CorrectionSession:
       ```python
       session = CorrectionSession(
           outcome=outcome,
           project_id=project_id,
           proposal_set=proposal_set,
           best_proposal_set=proposal_set,
       ```

    4. Save pending proposals: `save_pending_proposals(project_id, proposal_set.to_dict())`

    5. Enter interactive loop:
       ```
       while True:
           if session.cycle_limit_reached:
               print cycle limit message: "I've refined this {session.max_cycles} times. Here's the best I achieved:"
               render best_proposal_set
               prompt = "Approve [N] or edit [N]: "
           else:
               prompt = "Feedback, 'edit [N]', 'approve [N]', or 'quit': "

           user_input = input(prompt).strip()

           if user_input.lower() == 'quit':
               save_pending_proposals(project_id, session.proposal_set.to_dict())
               print("Proposals saved. Resume with: openclaw-approve")
               return 0

           if user_input.lower().startswith('approve'):
               # Parse archetype number
               selected = _parse_selection(user_input, session.proposal_set)
               if selected is None: print error; continue
               # Compute pushback
               pushback = ""
               if selected.rubric_score:
                   # Find original score for this archetype
                   original = _find_original_proposal(session, selected.archetype)
                   if original and original.rubric_score:
                       pushback = compute_pushback_note(original.rubric_score, selected.topology, weights, pushback_threshold)
               # Approve
               entry = approve_topology(project_id, selected.topology, correction_type, pushback)
               if pushback:
                   print(f"\n{Colors.YELLOW}{pushback}{Colors.RESET}\n")
               print(f"{Colors.GREEN}Topology approved and saved.{Colors.RESET}")
               return 0

           if user_input.lower().startswith('edit'):
               # Parse optional number
               selected = _parse_selection(user_input, session.proposal_set)
               if selected is None: selected = session.proposal_set.proposals[0]
               draft_path = export_draft(selected, project_id)
               print(f"Draft exported to: {draft_path}")
               print("Edit the file, then type 'done' to import or 'cancel':")
               resp = input("> ").strip().lower()
               if resp == 'done':
                   try:
                       graph, lint_result = import_draft(project_id, registry, max_concurrent)
                   except ValueError as e:
                       print(f"{Colors.RED}Import error: {e}{Colors.RESET}")
                       print("Your file has been left in place. Fix the error and try again.")
                       continue
                   if not lint_result.valid:
                       print(f"{Colors.RED}Blocked: {lint_result.rejected_roles}{Colors.RESET}")
                       print("Your file has been left in place.")
                       continue
                   if lint_result.adjusted:
                       for adj in lint_result.adjustments:
                           print(f"{Colors.YELLOW}  Adjusted: {adj}{Colors.RESET}")
                   # Show diff
                   diff_summary = render_diff_summary(selected, ...)
                   print(diff_summary)
                   # Approve the edited topology
                   pushback = compute_pushback_note(...) if selected.rubric_score else ""
                   entry = approve_topology(project_id, graph, "hard", pushback)
                   if pushback:
                       print(f"\n{Colors.YELLOW}{pushback}{Colors.RESET}\n")
                   print(f"{Colors.GREEN}Edited topology approved.{Colors.RESET}")
                   return 0
               else:
                   print("Edit cancelled.")
                   continue

           # Treat as feedback (soft correction)
           if session.cycle_limit_reached:
               print("Cycle limit reached. Use 'approve [N]' or 'edit [N]'.")
               continue
           old_proposals = session.proposal_set
           try:
               new_set = apply_soft_correction(session, user_input, weights)
           except Exception as e:
               print(f"{Colors.RED}Re-proposal error: {e}{Colors.RESET}")
               continue
           # Show diff for each archetype
           for old_p, new_p in zip(old_proposals.proposals, new_set.proposals):
               if old_p.archetype == new_p.archetype:
                   print(render_diff_summary(old_p, new_p))
           # Show full updated output
           print(render_full_output(new_set, threshold))
           save_pending_proposals(project_id, new_set.to_dict())
       ```

    6. Helper functions:
       - `_parse_selection(input_str, proposal_set) -> Optional[TopologyProposal]`: parse "approve 2" or "edit lean" and return the matching proposal. Support both index (1-based) and archetype name.
       - `_find_original_proposal(session, archetype)`: find the original proposal for pushback comparison (from session's first proposal_set in correction_history, or the initial best_proposal_set).

    7. Determine correction_type for approve: "initial" if cycle_count == 0, "soft" if corrections happened but no edit, "hard" if edit flow used.

    8. Add `--edit` flag to argparse for direct editor launch (subprocess opens $EDITOR on the draft file). This is a convenience flag per user decision.
