2. Create `workspace/occc/src/app/api/memory/route.ts` with GET handler:
   - Read `openclaw.json` via `readOpenClawConfig()` to get memU URL from `config.memory?.memu_api_url` (default `http://localhost:18791`)
   - Extract `project` and `search` from query params; fall back to `getActiveProjectId()` if no project param
   - If `search` param present: POST to `${memuUrl}/retrieve` with `{ queries: [{ role: 'user', content: search }], where: { user_id: projectId } }` — normalize response (may be array or `{ items: [] }`)
   - If no `search` param: GET `${memuUrl}/memories?user_id=${projectId}` — normalize response (may be raw array or `{ items: [] }`)
   - Return `{ items, total: items.length, projectId, mode: search ? 'search' : 'browse' }`
   - Wrap in try/catch, return 500 on error

3. Create `workspace/occc/src/app/api/memory/[id]/route.ts` with DELETE handler:
   - Read memU URL from config (same pattern as GET route)
   - Extract `id` from route params
   - Forward DELETE to `${memuUrl}/memories/${id}`
   - Return the memU response JSON on success, or `{ error: 'Delete failed' }` with the upstream status code on failure
   - Wrap in try/catch, return 500 on error

CRITICAL: Normalize memU responses in both GET cases. `GET /memories` returns a plain JSON array, not `{ items: [] }`. `POST /retrieve` response shape varies. Use: `const items = Array.isArray(data) ? data : (data.items ?? []);`
