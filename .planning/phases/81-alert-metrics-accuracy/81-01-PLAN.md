<objective>
Fix two integration gaps (GAP-03, GAP-04) that prevent autonomy alerts from appearing in the per-project dashboard feed and cause the metrics endpoint to always report max_concurrent=3 regardless of project config.

Purpose: Autonomy escalation alerts are currently silently dropped by useLiveEvents.ts because AutonomyEventBus emits project_id='unknown'. The metrics endpoint reports hardcoded pool capacity. Both are surgical, independent bug fixes.

Output: events.py with project_id field, hooks.py threading project_id and emitting bridgeable 'autonomy.escalation' events, metrics.py reading per-project max_concurrent, and new/extended unit tests for all fixed behaviors.
</objective>
