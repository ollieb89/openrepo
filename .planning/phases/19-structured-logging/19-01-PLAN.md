  <action>
Create `orchestration/logging.py` — the shared structured logging module for all orchestration components. Use Python stdlib `logging` only (no external deps).

**JSON Formatter class (`StructuredFormatter`):**
- Subclass `logging.Formatter`
- Override `format()` to emit a single-line JSON object with these fields:
  - `timestamp`: ISO 8601 format (e.g., `2026-02-24T14:30:00.123456`)
  - `level`: log level name (DEBUG, INFO, WARNING, ERROR)
  - `component`: the component name passed to `get_logger()`
  - `message`: the formatted log message
  - `task_id`: from `record.__dict__` extra field (if present, else omit key)
  - `project_id`: from `record.__dict__` extra field (if present, else omit key)
  - Any additional extra keys from `record.__dict__` that are not standard LogRecord attributes (to support ad-hoc context like `exit_code`, `retry_count`, etc.)
- Use `json.dumps(ensure_ascii=False)` for output — one JSON object per line, no pretty-printing

**`get_logger(component: str)` factory function:**
- Returns a `logging.Logger` named `openclaw.{component}`
- Attaches a `logging.StreamHandler(sys.stderr)` with the `StructuredFormatter`
- Sets log level from `orchestration.config.LOG_LEVEL`
- Uses a module-level set to track already-configured loggers and avoid adding duplicate handlers on repeated calls
- Returns the same logger instance on repeated calls for the same component name

**Update `orchestration/config.py`:**
- Add `LOG_LEVEL` constant: `LOG_LEVEL = os.environ.get("OPENCLAW_LOG_LEVEL", "INFO").upper()`
- Import `os` at top of file

**Update `orchestration/__init__.py`:**
- Add `from .logging import get_logger` import
- Add `'get_logger'` to `__all__` list under a "Logging" comment section
