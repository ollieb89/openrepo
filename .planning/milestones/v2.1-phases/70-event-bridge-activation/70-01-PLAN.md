<objective>
Activate the event bridge: wire event_bus to the Unix socket transport so all published events flow to connected clients, and auto-start the socket server during long-running orchestration commands.

Purpose: The event infrastructure (protocol, transport, event_bus) was built in earlier phases but never activated — events published via event_bus.emit() go nowhere, and state_engine/hooks bypass event_bus entirely by calling event_bridge.publish() directly. This plan unifies all event publishing through event_bus.emit() and bridges events to the Unix socket transport for real-time consumption by the dashboard SSE endpoint.

Output: A single event path (event_bus.emit -> bridge handler -> Unix socket -> connected clients) that starts automatically and degrades gracefully.
</objective>

**2. Create bridge.py:**
Create `packages/orchestration/src/openclaw/events/bridge.py` with:

- `_bridge_running: bool = False` module-level flag (true singleton check).
- `_loop: Optional[asyncio.AbstractEventLoop] = None` — the bridge's own event loop running in a daemon thread.

- `def _envelope_to_event(envelope: Dict[str, Any]) -> OrchestratorEvent`:
  Maps event_bus envelope dict keys to OrchestratorEvent fields. The envelope has `event_type` (string like "task.started"), `project_id`, `task_id`, `agent_id`, etc. Map `event_type` to the matching `EventType` enum value. Determine `EventDomain` from the event_type prefix (e.g., "task.*" -> TASK, "agent.*" -> AGENT, "autonomy.*" -> AUTONOMY, "memory.*" -> MEMORY, "pool.*" -> POOL). If event_type doesn't match any known EventType, return None (skip unknown events). Remaining keys become the `payload` dict (exclude event_type, project_id, task_id, agent_id, correlation_id, timestamp from payload — those are top-level fields).

- `def _bridge_handler(envelope: Dict[str, Any]) -> None`:
  Called by event_bus in a daemon thread. Converts envelope to OrchestratorEvent via `_envelope_to_event()`. If conversion returns None, log debug and return. Otherwise, schedule `event_bridge.publish(event)` on the bridge's asyncio loop using `asyncio.run_coroutine_threadsafe(_loop, ...)`. Wrap everything in try/except — failures log warning, never raise (fire-and-forget per user decision).

- `def ensure_event_bridge() -> bool`:
  Idempotent startup function. If `_bridge_running` is True, return True immediately. Otherwise:
  1. Create a new asyncio event loop.
  2. Start a daemon thread running `loop.run_forever()`.
  3. Schedule `event_bridge.start_server()` on that loop (using `asyncio.run_coroutine_threadsafe`), wait for result with 5s timeout.
  4. If server started successfully: subscribe a wildcard handler to event_bus — use `event_bus.subscribe("*", _bridge_handler)` — BUT event_bus only matches exact event_type strings. Instead, register `_bridge_handler` for every known EventType value: iterate `EventType` enum members and call `event_bus.subscribe(member.value, _bridge_handler)` for each.
  5. Set `_bridge_running = True`, return True.
  6. On ANY exception: log warning "Event bridge failed to start — events will not be forwarded", set `_bridge_running = False`, return False. Orchestration continues without events per user decision.
