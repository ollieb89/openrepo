<objective>
Add a recovery scan at pool startup that detects orphaned tasks and applies a configurable per-project recovery policy, plus the recovery_policy schema extension in project_config.py.

Purpose: After a crash or ungraceful shutdown, tasks may be stuck in `in_progress`, `interrupted`, or `starting` states. Without recovery, these tasks remain stuck forever. This plan adds automatic recovery on pool startup with three configurable policies: mark_failed (default), auto_retry (with safety checks), and manual (leave for operator).

Output: `run_recovery_scan()` method on L3ContainerPool, `recovery_policy` in get_pool_config(), and comprehensive tests.

**Part B: pool.py — Add run_recovery_scan() method**

Add `run_recovery_scan()` as an async method on `L3ContainerPool`. Place it after `drain_pending_memorize_tasks()` (added in plan 02).

```python
async def run_recovery_scan(self) -> dict:
    """Scan for orphaned tasks at pool startup and apply recovery policy.

    Checks workspace-state.json for tasks in 'in_progress', 'interrupted',
    or 'starting' states that are older than their skill timeout. Applies
    the configured recovery_policy (mark_failed / auto_retry / manual).

    Returns:
        Summary dict with scanned, mark_failed, retried, manual counts.
    """
```

Implementation details:
- Get `recovery_policy` from `self._pool_config.get("recovery_policy", "mark_failed")`
- Create `JarvisState(get_state_path(self.project_id))`
- Call `jarvis.list_active_tasks()` to get non-terminal task IDs
- For each task:
  - Read task via `jarvis.read_task(task_id)`
  - Get `status` from task dict
  - Only process tasks with status in `{"in_progress", "interrupted", "starting"}`
  - Get `skill_hint` from `task.get("skill_hint", "code")`
  - Get `timeout_s` from `get_skill_timeout(skill_hint)`
  - Get `spawn_requested_at` from `task.get("metadata", {}).get("spawn_requested_at")`
  - If `spawn_requested_at` is None or not a number: treat as expired. Set `age_s = timeout_s + 1`. Log a warning about missing timestamp.
  - Otherwise: `age_s = time.time() - spawn_requested_at`
  - If `age_s < timeout_s`: skip (task is still within its allowed time window)
  - Apply policy:
    - `mark_failed`: `jarvis.update_task(task_id, "failed", f"RECOVERED: task-{task_id} -> mark_failed (age: {age_s:.0f}s, timeout: {timeout_s}s)")`
    - `auto_retry`:
      - Check `retry_count = task.get("metadata", {}).get("retry_count", 0)` (or top-level if stored there by pool.py metrics)
      - If `retry_count >= 1`: fall back to `mark_failed` with message `"RECOVERED: task-{task_id} -> mark_failed (retry limit reached)"`
      - Otherwise: check for existing commits on staging branch `l3/task-{task_id}` using subprocess:
        ```python
        import subprocess
        workspace = get_workspace_path(self.project_id)
        branch = f"l3/task-{task_id}"
        try:
            result = subprocess.run(
                ["git", "log", "--oneline", f"HEAD..{branch}"],
                capture_output=True, text=True, timeout=5,
                cwd=workspace,
            )
            has_commits = bool(result.stdout.strip())
        except Exception:
            has_commits = True  # conservative: assume commits exist on error
        ```
        If `has_commits`: fall back to `mark_failed` with message `"RECOVERED: task-{task_id} -> mark_failed (partial commits on staging branch)"`
        If no commits: mark for retry (increment counter, leave in `starting` state for re-spawn — or mark as `pending_retry` which pool's main loop will pick up). For simplicity in this phase, log the retry intent and mark as `failed` with message `"RECOVERED: task-{task_id} -> auto_retry (queued for re-spawn)"`. Actual re-spawn scheduling is out of scope — the recovery scan identifies and flags; the operator or next pool invocation acts on it.
    - `manual`: log but do NOT change state. Just count for the summary.
- Always log startup summary:
```python
logger.info(
    "Pool startup: recovery scan complete",
    extra={
        "project_id": self.project_id,
        "scanned": scanned,
        "mark_failed": mark_failed_count,
        "retried": retried_count,
        "manual": manual_count,
        "policy": recovery_policy,
    },
)
```
Format matches CONTEXT.md: `Pool startup: scanned N tasks, M interrupted -> mark_failed, K retried`

Return `{"scanned": scanned, "mark_failed": mark_failed_count, "retried": retried_count, "manual": manual_count}`.

Also update `_POOL_DEFAULTS` at the top of pool.py to include `"recovery_policy": "mark_failed"` for consistency.
