# Milestone: v1.6 Agent Autonomy

**Goal**: Shift OpenClaw from a purely reactive specialist swarm to a more autonomous agent that can decompose ambiguous directives, monitor its own progress, and escalate logic only when confidence is low.

## Phases

### Phase 54: Autonomy Framework Formulation ✓
**Status**: Implemented (4/4 plans complete) — tests pending
**Goal**: Define the metadata schema for "Autonomy" (confidence scores, retry budgets, escalation triggers) and hook them into the `spawn` logic.
**Depends on**: Milestone v1.5 (shipped)
**Requirements**: AUTO-02, AUTO-04, AUTO-05
**Deliverables**:
- `packages/orchestration/src/openclaw/autonomy/` — full package with types, state machine, confidence, events, hooks, client, memory
- 4-state machine: PLANNING → EXECUTING → BLOCKED → COMPLETE/ESCALATING
- ConfidenceScorer with ThresholdBasedScorer implementation
- Event bus with debouncing for confidence updates
- L3 AutonomyClient with HTTP + sentinel file fallback
**Success Criteria**:
- [x] L3 agents receive an `autonomy_config` block in their SOUL (via hooks)
- [x] State machine validates all transitions with retry logic
- [x] Events emitted on state changes
- [ ] All tests passing (pending TEST-01)
