    **Dockerfile** - L3 specialist container image:
    - Base image: `debian:bookworm-slim` (matches existing sandbox image pattern from containers.json)
    - Install: git, python3, python3-pip, curl, jq (essentials for code/test tasks)
    - Install: the orchestration package (copy from host, pip install)
    - Do NOT bake in any specific CLI runtime (Claude Code/Codex/Gemini). The entrypoint will check CLI_RUNTIME env var and use whatever binary is mounted or available.

    **entrypoint.sh** - Container startup script:
    ```bash
    #!/usr/bin/env bash
    set -euo pipefail

    # Required environment variables (set by spawner)
    : "${TASK_ID:?TASK_ID is required}"
    : "${SKILL_HINT:?SKILL_HINT is required}"
    : "${STAGING_BRANCH:?STAGING_BRANCH is required}"
    : "${CLI_RUNTIME:=claude-code}"

    STATE_FILE="/workspace/.openclaw/workspace-state.json"

    # Helper: update state.json via Python state engine
    update_state() {
      local status="$1"
      local message="$2"
      python3 -c "
    import sys; sys.path.insert(0, '/orchestration')
    from state_engine import JarvisState
    js = JarvisState('${STATE_FILE}')
    js.update_task('${TASK_ID}', '${status}', '${message}')
    "
    }

    # 1. Report startup
    update_state "starting" "Container initialized. Skill: ${SKILL_HINT}, Runtime: ${CLI_RUNTIME}"

    # 2. Create staging branch
    cd /workspace
    git checkout -b "${STAGING_BRANCH}" main 2>/dev/null || git checkout "${STAGING_BRANCH}"
    update_state "in_progress" "Checked out staging branch: ${STAGING_BRANCH}"

    # 3. Execute task based on skill hint
    # The actual task description comes via TASK_DESCRIPTION env var
    # The CLI runtime executes it
    update_state "in_progress" "Executing task with ${CLI_RUNTIME}..."

    # Placeholder: actual CLI invocation will depend on runtime
    # This is the hook point where Claude Code / Codex / Gemini CLI runs
    if command -v "${CLI_RUNTIME}" &>/dev/null; then
      ${CLI_RUNTIME} --task "${TASK_DESCRIPTION:-No task description}" 2>&1 | tee /tmp/task-output.log
      EXIT_CODE=$?
    else
      echo "WARNING: CLI runtime '${CLI_RUNTIME}' not found. Running in dry-run mode."
      EXIT_CODE=0
    fi

    # 4. Capture results
    if [ $EXIT_CODE -eq 0 ]; then
      # Stage and commit changes on staging branch
      git add -A
      if git diff --cached --quiet; then
        update_state "completed" "Task completed. No files changed."
      else
        CHANGED_FILES=$(git diff --cached --name-only | head -20)
        git commit -m "L3 task ${TASK_ID}: ${SKILL_HINT}" --allow-empty
        update_state "completed" "Task completed. Changed files: ${CHANGED_FILES}"
      fi
    else
      # Capture failure context
      LAST_OUTPUT=$(tail -50 /tmp/task-output.log 2>/dev/null || echo "No output captured")
      update_state "failed" "Task failed (exit code ${EXIT_CODE}). Last output: ${LAST_OUTPUT}"
      exit $EXIT_CODE
    fi
    ```

    Make entrypoint.sh executable. Ensure git user config is set inside the container (git config user.name "L3 Specialist" and git config user.email "l3@openclaw.local") early in the entrypoint.

    Do NOT install Docker inside the L3 container (anti-pattern: Docker-in-Docker). L3 containers are managed by L2 from the host.

    ```json
            "requires_gpu": { "type": "boolean", "default": false },
            "cli_runtime": { "type": "string", "default": "claude-code" }
          },
          "handler": "python3 spawn.py"
        }
    ```

    **requirements.txt**:
    ```
    docker>=7.1.0
    ```

    **spawn.py** - Container spawning module:
    - Import docker, docker.types.DeviceRequest
    - Import JarvisState from orchestration.state_engine (add orchestration to sys.path)
    - Function `spawn_l3_specialist(task_id, skill_hint, task_description, workspace_path, requires_gpu=False, cli_runtime='claude-code')`:
      - Create staging branch name: `l3/task-{task_id}`
      - Build container config dict:
        - image: `openclaw-l3-specialist:latest`
        - name: `openclaw-l3-{task_id}`
        - detach: True
        - volumes: Full workspace mount at /workspace (mode: 'rw'), orchestration package mount at /orchestration (mode: 'ro')
        - environment: TASK_ID, SKILL_HINT, STAGING_BRANCH, CLI_RUNTIME, TASK_DESCRIPTION
        - security_opt: ['no-new-privileges'] (per HIE-04 requirement)
        - cap_drop: ['ALL']
        - mem_limit: '4g'
        - cpu_quota: 100000 (1 CPU)
        - restart_policy: {'Name': 'no'} (L2 handles retries, not Docker)
        - labels: openclaw.tier=l3, openclaw.task_id={task_id}, openclaw.spawned_by=pumplai_pm, openclaw.skill={skill_hint}
        - user: match host UID/GID to avoid permission errors (use os.getuid())
      - If requires_gpu: add device_requests with DeviceRequest(count=-1, capabilities=[['gpu']], driver='nvidia')
      - Load L3 config from agents/l3_specialist/config.json to get timeout for the skill
      - Create task entry in state.json via JarvisState.create_task()
      - Spawn container via client.containers.run(**config)
      - Return container object and task metadata
    - Function `cleanup_container(container)`: Force remove container, handle already-removed case gracefully.
    - Function `get_container_logs(container, tail=100)`: Get last N log lines from container.

    **pool.py** - Container pool management:
    - Import asyncio, spawn_l3_specialist from spawn.py, JarvisState
    - Class `L3ContainerPool`:
      - `__init__(max_concurrent=3)` — asyncio.Semaphore(3), active_containers dict
      - `async spawn_and_monitor(task_id, skill_hint, task_description, workspace_path, requires_gpu=False, cli_runtime='claude-code')`:
        - Acquire semaphore (blocks if 3 containers already running)
        - Call spawn_l3_specialist (sync, run in executor)
        - Monitor container with configurable timeout (from skill registry: 600s for code, 300s for test)
        - On container exit: check exit code
        - If failed and retry_count == 0: remove container, spawn again (retry once per locked decision)
        - If failed after retry: update state to "failed" with error context (exit code, last 50 log lines, modified files)
        - On success: update state to "completed"
        - In finally block: remove container (ephemeral lifecycle), release semaphore
      - `async monitor_container(container, task_id, timeout_seconds)`:
        - Use container.wait(timeout=timeout_seconds) in executor
        - Stream logs during wait for real-time monitoring
        - Return exit code and status
      - `get_active_count()` — Return number of active containers
      - `list_active()` — Return list of active task IDs

    Use `asyncio.get_event_loop().run_in_executor(None, ...)` to wrap synchronous docker-py calls in async context. This allows the pool to manage multiple containers concurrently without blocking.

    Handle edge cases:
    - Docker daemon not running → raise clear error
    - Image not found → raise clear error with build instructions
    - Container name conflict (already exists) → remove old container first
    - Timeout → kill container, update state to "timeout"
