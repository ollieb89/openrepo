Add config-driven per-project concurrency limits to the L3 container pool, with hot-reload semantics and graceful validation.

Purpose: Currently max_concurrent is hardcoded to 3 for all projects. This plan makes it configurable per project via l3_overrides in project.json, with fresh reads on every spawn for hot-reload, and validation that falls back to defaults on bad input.

Output: PoolRegistry reads pool config from project.json on every get_pool() call, recreates semaphores when max_concurrent changes, and validates config at load time.
