**`service.py`**: MemUService initialization:
```python
from memu import MemUService
from .config import Settings

async def init_service(settings: Settings) -> MemUService:
    service = MemUService(
        llm_profiles={
            "default": {
                "api_key": settings.OPENAI_API_KEY,
                "chat_model": settings.OPENAI_CHAT_MODEL,
            },
            "embedding": {
                "api_key": settings.OPENAI_API_KEY,
                "embed_model": settings.OPENAI_EMBED_MODEL,
            },
        },
        database_config={
            "metadata_store": {
                "provider": "postgres",
                "ddl_mode": "create",
                "dsn": settings.dsn,
            },
            "vector_index": {
                "provider": "pgvector",
                "dsn": settings.dsn,
            },
        },
        memorize_config={
            "llm_temperature": 0.0,
        },
    )
    return service
```
Note: `ddl_mode='create'` auto-creates tables + vector extension on first connect. MemUService constructor may be synchronous — if so, do not `await` it, just call it directly inside the async lifespan. Wrap in try/except and log errors.

**`models.py`**: Pydantic request/response models:
- `MemorizeRequest`: `resource_url: str`, `modality: str = "conversation"`, `user: dict[str, Any] | None = None`
- `RetrieveRequest`: `queries: list[dict[str, Any]]`, `where: dict[str, Any] | None = None`
- `MemorizeAccepted`: `status: str = "accepted"`, `message: str = "Memorization queued"`
- `HealthResponse`: `status: str`, `service: str`, `memu_initialized: bool`

**`main.py`**: FastAPI app with lifespan:
```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from .config import Settings
from .service import init_service
from .logging import get_logger
from .routers import health, memorize, retrieve, memories

logger = get_logger("main")

@asynccontextmanager
async def lifespan(app: FastAPI):
    settings = Settings()
    logger.info("Initializing MemUService...")
    try:
        app.state.memu = await init_service(settings)  # or sync call if constructor is sync
        logger.info("MemUService initialized successfully")
    except Exception as e:
        logger.error(f"Failed to initialize MemUService: {e}")
        app.state.memu = None  # Health endpoint will report uninitialized
    yield

app = FastAPI(title="OpenClaw Memory Service", lifespan=lifespan)
app.include_router(health.router)
app.include_router(memorize.router)
app.include_router(retrieve.router)
app.include_router(memories.router)
```

If MemUService.__init__ is synchronous (not a coroutine), just call it without await. Check at runtime.

**`routers/__init__.py`**: Empty.

**`routers/health.py`**: GET /health — returns 200 with `{"status": "ok", "service": "openclaw-memory", "memu_initialized": true/false}`. Check `request.app.state.memu is not None`.

**`routers/memorize.py`**: POST /memorize — accepts `MemorizeRequest`, adds `_run_memorize` as BackgroundTask, returns 202 Accepted with `MemorizeAccepted` body. The background task calls `service.memorize(resource_url=..., modality=..., user=...)`. Wrap background task in try/except and log errors — do not let extraction failures crash the worker.

**`routers/retrieve.py`**: POST /retrieve — accepts `RetrieveRequest`, calls `await service.retrieve(queries=..., where=...)`, returns the result directly. This is synchronous from the caller's perspective (blocks until retrieval completes).

**`routers/memories.py`**: Two endpoints:
- GET /memories — optional query param `user_id: str | None = None`. Calls `await service.list_memory_items(where={"user_id": user_id} if user_id else None)`. Returns list.
- DELETE /memories/{memory_id} — calls `await service.delete_memory_item(memory_id=memory_id)`. Returns `{"status": "deleted", "memory_id": memory_id}`.

All router endpoints access the service via `request.app.state.memu`. If memu is None (init failed), return 503 Service Unavailable.
