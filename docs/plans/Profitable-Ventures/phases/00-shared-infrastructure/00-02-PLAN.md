# Plan: Phase 0, Wave 0.2 - Database & Auth

<context>
**Phase:** 0 - Shared Infrastructure
**Wave:** 0.2
**Goal:** Multi-tenant data layer and JWT auth. Every API endpoint needs this.
**Context:** Hybrid Sharing (system data is global, user data strictly scoped). Rate limiting uses Rich Headers & Custom Codes with Redis Backed storage. Terraform schema bootstrapping is heavily utilized, so Alembic migrations must be crisp and automated.
</context>

<requirements>
- SQLAlchemy 2.0 async models: `Organization`, `User`, `ApiKey` with `org_id` FK.
- Alembic migration setup + initial migration.
- JWT auth middleware (RS256, 15min access, 7d refresh, org_id in claims).
- Auth API routes (register, login, refresh, API key CRUD) with bcrypt.
- Rate Limit Strategy: Redis Backed, Rich Headers (Retry-After) & Custom Codes.
- Tenant isolation middleware (Auto-filter by `org_id` from JWT).
- Gate G0.2: Authenticated request with org-scoped data isolation proven in test.
</requirements>
