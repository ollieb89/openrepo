**Create `src/components/ProjectSelector.tsx`:**

A compact dropdown for the dashboard header. Per user decisions:
- Each project entry shows name + colored status badge (active/idle/error)
- Uses native `<select>` element styled with Tailwind (per research: avoid Radix/shadcn overhead)
- `ChevronDown` icon from `lucide-react` overlaid on the select

Props interface:
```typescript
interface ProjectSelectorProps {
  projects: Array<{ id: string; name: string; status: 'active' | 'idle' | 'error' }>;
  selectedId: string;
  onSelect: (id: string) => void;
  isLoading?: boolean;  // show shimmer during project list fetch
}
```

Status badge colors (Tailwind):
- `active`: `bg-teal-100 text-teal-700`
- `idle`: `bg-slate-100 text-slate-600`
- `error`: `bg-red-100 text-red-700`

The badge sits to the left of the select element showing the currently selected project's status. The select itself has compact styling: `text-sm font-medium`, white background, slate border, teal focus ring. When `isLoading` is true, show a shimmer placeholder instead of the dropdown.

Mark as `'use client'`.

**Modify `src/hooks/useSwarmState.ts`:**

1. Change the function signature from `useSwarmState()` to `useSwarmState(projectId: string)`.
2. Build the SWR URL dynamically: `/api/swarm?project=${encodeURIComponent(projectId)}`. The SWR cache key IS the URL, so changing projectId automatically invalidates the cache — no manual cache clearing needed.
3. Build the SSE URL dynamically: `/api/swarm/stream?project=${encodeURIComponent(projectId)}`.
4. Add `projectId` to the `connectSSE` useCallback dependency array. This causes the useEffect cleanup to fire (closing old EventSource) and reconnect with the new project's SSE URL on every project switch.
5. Do NOT change the return type — same `UseSwarmStateReturn` interface. Add `isValidating` to the return for the switching skeleton:
   - Import and destructure `isValidating` from the SWR hook.
   - Add `isValidating` to the return object.

The existing exponential backoff reconnection logic stays unchanged. The only changes are:
- Function accepts `projectId` parameter
- URLs include `?project=` query param
- `projectId` in dependency arrays
- `isValidating` exposed in return value
