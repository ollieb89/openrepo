  <name>Task 1: Pattern extraction engine</name>
  <files>orchestration/suggest.py</files>
  <action>
Create `orchestration/suggest.py` as a stdlib-only module (no imports outside `orchestration.*` and Python stdlib at module level). Structure:

**1. Constants**
```python
REJECTION_QUERY = "task failed rejected error agent mistake"
MIN_CLUSTER_SIZE = 3          # ADV-01 requirement
DEFAULT_LOOKBACK_DAYS = 30
MAX_CLUSTER_FRACTION = 0.5    # Discard clusters spanning >50% of memories (Pitfall 2)
DOMAIN_STOPWORDS = {          # Domain-specific stopwords beyond length filter
    "task", "agent", "error", "failed", "status", "completed",
    "the", "a", "an", "to", "in", "of", "and", "or", "was", "is",
    "that", "this", "with", "from", "for", "not", "but",
}
```

**2. `_suggestions_path(project_id: str) -> Path`**
- Returns `_find_project_root() / "workspace" / ".openclaw" / project_id / "soul-suggestions.json"`
- Import `_find_project_root` from `orchestration.project_config` inside function body (lazy import so module is testable without full orchestration stack)

**3. `_load_suggestions(project_id: str) -> dict`**
- Reads `_suggestions_path(project_id)` if it exists, parses JSON, returns dict
- Returns `{"version": "1.0", "last_run": None, "suggestions": []}` on FileNotFoundError or JSONDecodeError

**4. `_save_suggestions(project_id: str, new_suggestions: list[dict], dry_run: bool = False) -> None`**
- Loads existing via `_load_suggestions()`
- Merges: existing suggestions preserved, new ones appended (by id — no duplicates)
- Updates `last_run` to current ISO timestamp (`datetime.utcnow().isoformat() + "+00:00"`)
- If `dry_run=True`: prints JSON to stdout without writing
- If not dry_run: writes atomically to `_suggestions_path()` (mkdir parents, write to `.tmp`, rename)

**5. `_extract_keywords(text: str) -> list[str]`**
- Lowercase the text
- `re.findall(r"[a-z][a-z_-]+", text)` to get words
- Filter: `len(word) >= 4` AND `word not in DOMAIN_STOPWORDS`
- Return filtered list

**6. `_cluster_memories(memories: list[dict], lookback_days: int) -> dict[str, list[dict]]`**
- Compute `cutoff = time.time() - (lookback_days * 86400)`
- Filter `recent` using `m.get("created_at") or m.get("timestamp", 0) >= cutoff` (handle both int and float timestamps)
- Build `keyword_to_memories: dict[str, list[dict]] = defaultdict(list)`
- For each memory: extract content via `m.get("content", m.get("resource_url", ""))`, run `_extract_keywords()`, add memory to each keyword bucket
- Discard clusters where `len(mems) >= MAX_CLUSTER_FRACTION * len(recent)` (too generic)
- Return only clusters where `len(mems) >= MIN_CLUSTER_SIZE`

**7. `_fingerprint(keyword: str, evidence_count: int) -> str`**
- Returns `f"{keyword}:{evidence_count}"` — used for rejection suppression lookup

**8. `_should_suppress(keyword: str, cluster_size: int, existing: dict) -> bool`**
- Scans `existing["suggestions"]` for entries with `status == "rejected"` where fingerprint keyword matches `keyword`
- A rejected entry suppresses re-generation unless `cluster_size >= rejected_entry["suppressed_until_count"]`
- `suppressed_until_count` is stored at rejection time as `evidence_count * 2`
- Return True if any rejected entry triggers suppression

**9. `_build_suggestion(keyword: str, memories: list[dict]) -> dict`**
- Generate a stable id: `"sug-" + hashlib.md5(keyword.encode()).hexdigest()[:6]`
- `pattern_description`: `f"Agents frequently encounter '{keyword}'-related issues ({len(memories)} occurrences)"`
- `evidence_examples`: take up to 3 memories, each as `{"task_id": m.get("task_id", "unknown"), "excerpt": m.get("content", m.get("resource_url", ""))[:120]}`
- `diff_text`: generate a concrete `## BEHAVIORAL PROTOCOLS` amendment:
  ```
  ## BEHAVIORAL PROTOCOLS
  - **{keyword.replace('-', ' ').title()} Awareness:** Review task context for {keyword}-related edge cases before committing output. If uncertain, verify by running tests or checking prior task logs.
  ```
- Return full suggestion dict matching the RESEARCH.md schema (id, status="pending", created_at=time.time(), pattern_description, evidence_count, evidence_examples, diff_text, rejected_at=None, rejection_reason=None, suppressed_until_count=None, accepted_at=None)

**10. `async def run_extraction(project_id: str, memu_url: str, lookback_days: int = DEFAULT_LOOKBACK_DAYS) -> list[dict]`**
- Queries memU for rejection memories: `async with MemoryClient(memu_url, project_id, AgentType.L2_PM) as client: memories = await client.retrieve(REJECTION_QUERY)`
- Also reads `workspace-state.json` activity logs as supplementary corpus (per RESEARCH.md open question 1 — activity log is primary; memU is supplementary):
  - Import `get_state_path` from `orchestration.project_config` inside body
  - Load `get_state_path(project_id)` if it exists, parse `tasks` dict, collect all activity_log entries where `status` contains "failed" or "interrupted" — shape each as `{"content": entry.get("entry", ""), "created_at": entry.get("timestamp", 0)}`
  - Merge with memU memories (dedup by content hash)
- Run `_cluster_memories(all_memories, lookback_days)`
- Load existing suggestions for suppression check
- For each cluster: if `_should_suppress(keyword, cluster_size, existing)` → skip; else build suggestion
- Return list of new suggestion dicts (not including already-existing ones)

**11. CLI entry point** (at bottom, inside `if __name__ == "__main__":`)
- argparse: `--project` (optional, falls back to `get_active_project_id()`), `--dry-run` flag, `--lookback-days` (int, default 30)
- Get `memu_url` from `get_memu_config()["memu_api_url"]` — exit 1 with message if not configured
- `suggestions = asyncio.run(run_extraction(project_id, memu_url, args.lookback_days))`
- `_save_suggestions(project_id, suggestions, dry_run=args.dry_run)`
- Print count: `f"Saved {len(suggestions)} suggestion(s) to soul-suggestions.json"` (or `f"Dry run: {len(suggestions)} suggestion(s) found"`)

**Critical constraint:** `suggest.py` must NOT import `soul_renderer` write functions anywhere. The module boundary is the approval gate (ADV-06). If you feel the urge to import `write_soul` inside `suggest.py`, stop — that logic belongs in the accept API route (Plan 02).
