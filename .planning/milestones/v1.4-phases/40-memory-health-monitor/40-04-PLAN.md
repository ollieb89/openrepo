  <files>workspace/occc/src/components/memory/MemoryPanel.tsx</files>
  <action>
    In `handleArchiveMemory` (lines 294-304), the function currently sends:
    ```
    body: JSON.stringify({ archived_at: new Date().toISOString() })
    ```

    This breaks because `MemoryUpdateRequest` requires `content: str` (non-optional).

    Fix: Look up the memory item from the SWR data cache (`data.items`) to get its current content, then send the content with an `[ARCHIVED]` prefix:

    1. At the start of `handleArchiveMemory`, find the memory item from `data?.items`:
       ```ts
       const item = data?.items?.find((m: MemoryItem) => m.id === memoryId);
       const currentContent = item?.content ?? '';
       ```

    2. Replace the PUT body from `{ archived_at: new Date().toISOString() }` to:
       ```ts
       { content: `[ARCHIVED ${new Date().toISOString()}] ${currentContent}` }
       ```

    This uses the existing working PUT → memU content-update pipeline. The `[ARCHIVED <timestamp>]` prefix serves as a soft-delete marker that is recoverable (operator can edit the content back). No backend changes required — `memu.update_memory_item()` only accepts `memory_id` + `memory_content`, so metadata fields would be ignored anyway.

    Do NOT add `archived_at` to `MemoryUpdateRequest` — the memU `update_memory_item()` API has no metadata passthrough, so backend changes would be dead code.
  </action>
