## FR-01: Tech Debt Resolution

### Task 1.1: Fix Async Event Loop Patterns

#### Problem
`asyncio.get_event_loop()` raises `RuntimeError` when called from a thread that already has an event loop running (common in tests and async contexts).

#### Solution Strategy
Replace `asyncio.get_event_loop()` with `asyncio.get_running_loop()` where appropriate, or use proper async patterns.

#### Files to Modify

**1. `packages/orchestration/src/openclaw/autonomy/runner.py` (Line 255)**
```python
# Before
loop = asyncio.get_event_loop()
output = await loop.run_in_executor(None, self._invoke_cli, prompt)

# After  
output = await asyncio.get_running_loop().run_in_executor(None, self._invoke_cli, prompt)
```

**2. `packages/orchestration/src/openclaw/events/bridge.py` (Line 133)**
```python
# Before
loop = asyncio.new_event_loop()

# After — use existing pattern from line 144:
# asyncio.run_coroutine_threadsafe(event_bridge.start_server(), loop)
# Keep as-is since it explicitly needs a new loop for the bridge thread
```

**3. `skills/spawn/pool.py` (Lines 242, 423, 699, 729)**
```python
# Before
loop = asyncio.get_event_loop()

# After
loop = asyncio.get_running_loop()  # Or use asyncio.create_future() directly
```

**4. `packages/orchestration/src/openclaw/state_engine.py` (Lines 39, 58)**
```python
# Before — runs in daemon thread via threading.Thread
def _run():
    try:
        asyncio.run(generate_memory_context(...))  # OK: new event loop
```
**Status: Already using `asyncio.run()` — no change needed**

#### Testing

# Run specific async-related tests
