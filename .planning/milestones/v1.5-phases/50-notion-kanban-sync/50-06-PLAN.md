Create `skills/notion-kanban-sync/reconcile_handler.py`:

1. `handle_reconcile(payload: dict, result: SyncResult) -> None`:
   - Set `client.bulk_mode = True` (enables rate-limit sleep between requests per Research pitfall 6).
   - Step 1: Read OpenClaw project states:
     - List project directories under `projects/` in OPENCLAW_ROOT.
     - For each, read `project.json` to get project_id, name, workspace.
     - Optionally read `.planning/ROADMAP.md` for current phase info.
   - Step 2: Fetch all Notion data:
     - Query Projects DB for all rows (paginate if > 100).
     - Query Cards DB for all active cards (Status != Archived, paginate).
   - Step 3: Build comparison sets:
     - OpenClaw projects set (by project_id).
     - Notion projects set (by OpenClaw ID property).
     - OpenClaw-linked Notion cards set (by openclaw_phase_id).

2. **Allowed corrections** (from SPEC reconcile flow):
   a. `_reconcile_missing_projects(openclaw_projects, notion_projects, result)`:
      - Projects in OpenClaw but not in Notion → create Projects DB row.
      - Use same logic as `_sync_project_registered` but without the triage card (only create the project row).
   b. `_reconcile_status_mismatch(openclaw_state, notion_cards, result)`:
      - For each OpenClaw-linked card, compare Status against expected state.
      - Expected state derived from OpenClaw: active phase → "In Progress", completed phase → "Done", etc.
      - If mismatch AND `_should_write_status(card)` → correct Notion to match OpenClaw.
   c. `_reconcile_missing_relations(notion_cards, notion_projects, result)`:
      - Cards with OpenClaw Phase ID containing a project_id but no Project relation → backfill relation.
   d. `_reconcile_dangling_cards(notion_cards, openclaw_phases, result)`:
      - Cards with OpenClaw Phase ID pointing to a phase that no longer exists in OpenClaw → set Status to "Archived".
      - "No longer exists" = project_id not in OpenClaw projects OR phase_id not in any project's ROADMAP.

3. **NOT allowed** (guards):
   - NEVER modify Status on unlinked cards (check `_should_write_status()`).
   - NEVER delete any Notion page (use Archived status).
   - NEVER modify Notion-owned fields (Priority, Notes, Target Week, Area on user-modified cards).

4. **Drift report** — return structured result:
   ```python
   result.extra = {
       "corrections_made": N,
       "in_sync_count": N,
       "archived_count": N,
   }
   ```

5. Wire into notion_sync.py dispatcher:
   ```python
   elif request_type == "reconcile":
       from reconcile_handler import handle_reconcile
       handle_reconcile(payload, result)
   ```

6. Set `client.bulk_mode = False` when reconcile completes (cleanup).

CRITICAL: Reconcile is the most API-intensive operation. Every query/update counts. Use bulk_mode with 0.35s sleep between requests. Log progress: "Reconciling project {N}/{total}".
