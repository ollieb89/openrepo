<task type="auto">
  <name>Task 1: Create openclaw-base Dockerfile and update Makefile</name>
  <files>docker/base/Dockerfile, Makefile</files>
  <action>
1. Create `docker/base/Dockerfile` with these exact contents, derived from `openclaw/Dockerfile.sandbox` but with the user named `sandbox` at UID 1000 (matching existing convention):

```dockerfile
FROM debian:bookworm-slim

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update \
  && apt-get install -y --no-install-recommends \
    bash \
    ca-certificates \
    curl \
    git \
    jq \
    python3 \
    ripgrep \
  && rm -rf /var/lib/apt/lists/*

RUN useradd --uid 1000 --create-home --shell /bin/bash sandbox
USER sandbox
WORKDIR /home/sandbox
```

Key differences from the original Dockerfile.sandbox:
- Explicitly set `--uid 1000` on useradd (was implicit, now explicit for clarity)
- No `CMD ["sleep", "infinity"]` — base image should not define a default command; consumers set their own entrypoint
- Add LABEL metadata (Claude's discretion): `LABEL openclaw.component="base"` and `LABEL openclaw.variant="bookworm-slim"`

2. Update `Makefile`:
- Add new target `docker-base` with comment `## Build openclaw base image`:
  `docker build -t openclaw-base:bookworm-slim docker/base/`
- Change `docker-l3` dependency from `docker-sandbox-base` to `docker-base`
- Update `docker-l3` help comment to say "depends on base image" instead of "depends on sandbox base"
- Add `docker-base` to the `.PHONY` declaration at the top
- Update `docker-all` to include `docker-base` instead of (or in addition to) `docker-sandbox-base` — keep old sandbox targets intact per user decision

Do NOT remove the existing `docker-sandbox-base` or `docker-sandbox-common` targets — they serve the submodule's own purposes per user decision.
