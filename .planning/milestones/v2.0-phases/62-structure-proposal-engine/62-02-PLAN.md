```python
# packages/orchestration/src/openclaw/topology/models.py (Plan 01 creates this)
from enum import Enum
from dataclasses import dataclass
from typing import Optional, List

class EdgeType(str, Enum):
    DELEGATION = "delegation"
    COORDINATION = "coordination"
    REVIEW_GATE = "review_gate"
    INFORMATION_FLOW = "information_flow"
    ESCALATION = "escalation"

@dataclass
class TopologyNode:
    id: str
    level: int        # 1, 2, 3
    intent: str
    risk_level: str   # "low", "medium", "high"
    resource_constraints: Optional[dict] = None
    estimated_load: Optional[float] = None

@dataclass
class TopologyEdge:
    from_role: str
    to_role: str
    edge_type: EdgeType

@dataclass
class TopologyGraph:
    nodes: List[TopologyNode]
    edges: List[TopologyEdge]
    project_id: str
    proposal_id: Optional[str] = None
    version: int = 1
    created_at: str = ""
    metadata: Optional[dict] = None
```

  <files>packages/orchestration/src/openclaw/topology/classifier.py, packages/orchestration/tests/test_topology_classifier.py</files>
  <behavior>
    - classify(graph: TopologyGraph) returns ArchetypeResult dataclass
    - ArchetypeResult has: archetype (str: "lean"/"balanced"/"robust"), confidence (float 0-1), explanation (str), traits (list[str])
    - Lean: linear chain or flat delegation, minimal coordination edges, no review gates
    - Balanced: tree with coordination edges, has review or escalation edges, moderate specialization
    - Robust: DAG with multiple coordination paths, review gates, fallback roles, redundancy
    - Classification is deterministic — same graph always gets same result
    - Edge cases get nearest archetype with explanation of atypical features
    - Traits provide annotations like "review-heavy coordination", "flat hierarchy"
  </behavior>
  <action>
Create `packages/orchestration/src/openclaw/topology/classifier.py`:

1. `@dataclass ArchetypeResult`:
   - `archetype: str` — "lean", "balanced", or "robust"
   - `confidence: float` — 0.0-1.0 how well it matches the archetype
   - `explanation: str` — why classified this way (per Phase 61 CONTEXT: always explain)
   - `traits: List[str]` — annotations like "review-heavy", "flat-hierarchy", "redundant-paths"

2. `class ArchetypeClassifier`:
   - `def classify(self, graph: TopologyGraph) -> ArchetypeResult`:
     Pattern-matching approach (per Phase 61 CONTEXT decision — NOT hard thresholds):

     a. Compute structural features:
        - `node_count = len(graph.nodes)`
        - `edge_count = len(graph.edges)`
        - `max_depth` = longest path in delegation edges
        - `coordination_edges` = count of EdgeType.COORDINATION
        - `review_gates` = count of EdgeType.REVIEW_GATE
        - `escalation_edges` = count of EdgeType.ESCALATION
        - `has_multiple_coordination_paths` = any node reachable via >1 coordination path

     b. Pattern match (per Phase 61 decisions):
        - **Lean**: `coordination_edges == 0 AND review_gates == 0 AND (max_depth <= 2 OR edge_count <= node_count)`
          Traits: "linear-chain" if max_depth == node_count-1, "flat-delegation" if max_depth <= 1
        - **Robust**: `review_gates >= 1 AND (escalation_edges >= 1 OR has_multiple_coordination_paths)`
          Traits: "review-heavy" if review_gates > 1, "redundant-paths" if has_multiple_paths, "fallback-roles" if escalation_edges > 0
        - **Balanced**: everything else (tree with moderate structure)
          Traits: "coordination-linked" if coordination_edges > 0, "review-gated" if review_gates > 0

     c. Confidence: based on how many pattern features match the archetype strongly.
        Strong match (all features) → 0.9. Partial match → 0.6-0.8.

     d. Explanation: f"Classified as {archetype} because {structural_reasons}. {'Atypical: ' + atypical if atypical else ''}"
