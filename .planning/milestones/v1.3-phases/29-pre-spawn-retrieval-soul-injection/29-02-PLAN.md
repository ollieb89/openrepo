**Test 4: `test_format_memory_context_empty_list`**
- Call `_format_memory_context([])`.

**Test 5: `test_format_memory_context_with_items`**
- Pass 3 memory items with `resource_url` content and one with `category: "l2_review"`.
- Assert result starts with `"## Memory Context\n\n"`.
- Assert each item appears as a bullet `"- {content} ({tag})"`.
- Assert the l2_review item has `(from L2 review)` tag.
- Assert others have `(from memory)` tag.

**Test 6: `test_format_memory_context_budget_enforcement`**
- Create items where each bullet is ~500 chars. Pass 6 items (total ~3000 chars).
- Assert the result length (total rendered string) is <= `MEMORY_CONTEXT_BUDGET` (2000).
- Assert at least 1 item was dropped (fewer bullets than input items).

- Assert result contains the original SOUL content.
- Assert result ends with the Memory Context section.
- Assert `"## Memory Context"` appears in the output.

**Test 8: `test_build_augmented_soul_empty_memory`**
- Same setup as Test 7 but pass `memory_context=""`.
- Assert result equals the base SOUL content (no Memory Context section appended).

**Test 9: `test_write_soul_tempfile_creates_and_returns_path`**
- Call `_write_soul_tempfile("test content")`.
- Assert returned path exists and its content equals `"test content"`.
- Clean up the temp file after assertion.

**Test 10: `test_format_memory_context_skips_empty_content_items`**
- Pass items where some have empty `resource_url` and empty `content`.
