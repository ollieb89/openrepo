  <name>Task 1: Fix SSE client payload handler in useSwarmState.ts (DSH-02)</name>
  <files>workspace/occc/src/hooks/useSwarmState.ts</files>
  <action>
In `workspace/occc/src/hooks/useSwarmState.ts`, replace the `onmessage` handler (lines 56-68) to make SSE the primary data path instead of a nudge-and-revalidate pattern.

**Current broken handler:**
```typescript
eventSource.onmessage = (event) => {
  if (!isMountedRef.current) return;
  try {
    const data = JSON.parse(event.data);
    if (data.updated) {
      mutate();
    }
  } catch {
    // Ignore parse errors for keepalive comments
  }
};
```

**Replace with:**
```typescript
eventSource.onmessage = (event) => {
  if (!isMountedRef.current) return;
  try {
    const parsed = JSON.parse(event.data) as SwarmStateResponse;
    if (parsed.agents) {
      // Full state payload received — inject directly into SWR cache
      // false = skip revalidation; SSE is the primary data source
      mutate(parsed, false);
      reconnectDelayRef.current = 1000; // reset backoff on successful data
    }
    // Error payloads (parsed.error) fall through without updating cache
  } catch {
    // Ignore parse errors for keepalive comments (": keepalive" is not JSON)
  }
};
```

**Key invariants to preserve:**
- `isMountedRef.current` guard stays (prevents state updates after unmount)
- `try/catch` around `JSON.parse` stays (SSE keepalive comments are not valid JSON)
- Exponential backoff reconnect logic on `onerror` stays unchanged
- SWR `refreshInterval: 2000` stays unchanged (fallback path)
- The `mutate` function reference from `useSWR` stays in the `useCallback` dependency array
- Do NOT change the `reconnectDelayRef.current = 1000` line at line 89 (after successful connection) — the new line inside `onmessage` resets backoff on successful DATA receipt, which is a different concern

**What NOT to do:**
- Do NOT remove the SWR fallback polling (`refreshInterval: 2000`)
- Do NOT use `mutate()` without `false` (that triggers a REST revalidation, defeating SSE)
- Do NOT change the outer `connectSSE` function structure or the `useCallback` deps
- Do NOT add new imports — `SwarmStateResponse` is already defined in the file (lines 8-13)
