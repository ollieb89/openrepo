  <action>
    Create the `orchestration/` Python package at the project root for Phase 3 shared modules.

    **orchestration/__init__.py** - Package init, export key classes.

    **orchestration/config.py** - Shared configuration:
    - STATE_FILE path: `workspace/.openclaw/workspace-state.json` (relative to project root)
    - LOCK_TIMEOUT: 5 seconds
    - LOCK_RETRY_ATTEMPTS: 3
    - POLL_INTERVAL: 1.0 second (per discretion: start with 1s polling)
    - SNAPSHOT_DIR: `workspace/.openclaw/snapshots/`

    **orchestration/state_engine.py** - The Jarvis Protocol state engine:
    - Class `JarvisState` with methods:
      - `__init__(state_file_path)` — Initialize with path to workspace-state.json
      - `update_task(task_id, status, activity_entry)` — Thread-safe state update using fcntl.flock(LOCK_EX). Read current state, update task entry (status + append to activity_log array with timestamp), write back atomically (seek(0), truncate, json.dump, flush). Release lock in finally block.
      - `read_state()` — Read current state with shared lock (LOCK_SH). Returns full state dict.
      - `read_task(task_id)` — Read single task entry. Returns task dict or None.
      - `create_task(task_id, skill_hint, metadata)` — Initialize a new task entry with status "pending", empty activity_log, created_at timestamp, skill_hint, and any metadata.
      - `list_active_tasks()` — Return tasks with status not in ("completed", "failed").
    - Use fcntl.flock() for all read/write operations. LOCK_EX for writes, LOCK_SH for reads.
    - Implement lock acquisition with timeout (try non-blocking LOCK_NB in loop with 0.1s sleep, timeout after LOCK_TIMEOUT seconds).
    - Full activity log: Every call to update_task appends a timestamped entry to the activity_log array (not just overwrites status). This is a locked decision — capture everything.
    - Atomic writes: seek(0), truncate(), json.dump(), flush() pattern inside the lock.
    - Handle edge cases: state file doesn't exist yet (create with empty schema), corrupt JSON (log error, reinitialize), lock timeout (raise TimeoutError).

    **workspace/.openclaw/workspace-state.json** - Initial empty state:
    ```json
    {
      "version": "1.0.0",
      "protocol": "jarvis",
      "tasks": {},
      "metadata": {
        "created_at": "<timestamp>",
        "last_updated": "<timestamp>"
      }
    }
    ```

    Follow Pattern 2 (State Synchronization) from 03-RESEARCH.md. Reference the atomic_state_update example for the lock-with-timeout pattern.

    Do NOT use threading or asyncio in the state engine itself — it should be a synchronous library usable from both sync and async callers. The async container pool (Plan 02) will call it from sync context within containers.
  </action>
