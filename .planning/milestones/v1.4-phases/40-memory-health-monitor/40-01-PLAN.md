Add Pydantic models to `docker/memory/memory_service/models.py`:

1. `HealthFlag(BaseModel)` — fields: `memory_id: str`, `flag_type: Literal["stale", "conflict"]`, `score: float` (age_score for stale as days/threshold, similarity for conflict), `recommendation: Literal["archive", "review", "merge"]`, `conflict_with: Optional[str] = None`
2. `HealthScanRequest(BaseModel)` — fields: `user_id: str` (REQUIRED, not optional — prevents cross-project scope leak per Pitfall 6), `age_threshold_days: int = 30`, `retrieval_window_days: int = 14`, `similarity_min: float = 0.75`, `similarity_max: float = 0.97`
3. `HealthScanResult(BaseModel)` — fields: `flags: list[HealthFlag]`, `scanned_at: str` (ISO timestamp), `totals: dict` (shape: `{"stale": N, "conflict": N, "total": N}`)
4. `MemoryUpdateRequest(BaseModel)` — fields: `content: str` (required, non-optional — prevents Pitfall 1 empty-body issue)

Add `run_health_scan()` function in `docker/memory/memory_service/service.py`:

**Staleness detection (`_check_staleness`):**
- Accepts a memory item, `age_threshold_days`, `retrieval_window_days`, and current UTC datetime
- Computes `item_age_days` from `item.created_at`
- If `item_age_days < age_threshold_days`: return None (not old enough)
- Check `extra.get("last_reinforced_at")` — parse with pendulum. If present and within `retrieval_window_days`: return None (retrieved recently)
- If `last_reinforced_at` absent: use `created_at` as proxy — if `item_age_days < retrieval_window_days`: return None (Pitfall 2: avoid false positives on new items without reinforcement data)
- Return `age_score = item_age_days / age_threshold_days`
- Recommend "archive" for score > 2.0, "review" otherwise

**Conflict detection (`_find_conflicts`):**
- Fetch all items once with `list_items(where={"user_id": user_id})`
- Build corpus of `(id, embedding)` tuples for items with non-None embeddings
- For each item, use `cosine_topk()` from `memu.database.inmemory.vector` to find top-10 neighbors
- Filter to similarity range `[similarity_min, similarity_max]` (exclude self-matches)
- Deduplicate pairs using `tuple(sorted([id_a, id_b]))` in a seen-set (Pitfall 3)
- Recommend "merge" for similarity > 0.90, "review" otherwise

**Orchestration (`run_health_scan`):**
- Takes `memu` service instance and `HealthScanRequest` params
- Calls `list_items(where={"user_id": user_id})` to get all items
- Runs staleness check on each item, collects `HealthFlag` objects
- Runs conflict detection on items with embeddings, collects `HealthFlag` objects
- Stores flags in `app.state.health_flags` dict keyed by memory_id (ephemeral, regenerated on each scan)
- Returns `HealthScanResult` with ISO `scanned_at` timestamp and totals dict

**Important:** Import `cosine_topk` from `memu.database.inmemory.vector`. If the import path differs in the installed package, check `workspace/memory/src/memu/database/inmemory/vector.py` for the correct function name and adjust.

Add two new endpoints to `docker/memory/memory_service/routers/memories.py`:

1. **POST /memories/health-scan** — follows existing GET/DELETE pattern in the file:
   - Get `memu` from `request.app.state.memu`, return 503 if None
   - Call `run_health_scan(memu, request.app, body)` passing the app for `app.state.health_flags` storage
   - Return `HealthScanResult` on success, 500 with detail on error
   - Log scan start/completion with structured logger

2. **PUT /memories/{memory_id}** — follows existing DELETE pattern:
   - Get `memu` from `request.app.state.memu`, return 503 if None
   - Call `memu.update_memory_item(memory_id=memory_id, memory_content=body.content)`
   - Return result on success
   - Return 404 on ValueError (item not found)
   - Return 500 on other exceptions

3. **GET /memories/health-flags** — lightweight endpoint to retrieve cached flags:
   - Read `request.app.state.health_flags` (default empty dict if not set)
   - Return the flags dict as JSON

Add imports for new models: `HealthScanRequest`, `MemoryUpdateRequest`, and the `run_health_scan` function from service.py.
