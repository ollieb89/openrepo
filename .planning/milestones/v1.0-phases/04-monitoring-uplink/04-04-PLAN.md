  <action>
**Update `next.config.ts`**:
- Add `serverExternalPackages: ['dockerode']` to the config (dockerode uses native Node.js bindings that cannot be bundled by Next.js)
- Keep existing `output: 'standalone'` setting
- Example:
  ```typescript
  const nextConfig: NextConfig = {
    output: 'standalone',
    serverExternalPackages: ['dockerode'],
  };
  ```

**Update `Dockerfile`** for Docker socket access (per research pitfall 2):

In the `deps` stage:
- Keep existing alpine base
- Ensure bun is available (or switch to npm if bun is not in node:22-alpine -- check and adapt)

In the `runner` stage:
- Install Docker CLI tools in the runner image for dockerode to function:
  ```dockerfile
  RUN apk add --no-cache docker-cli
  ```
- Add nextjs user to docker group for socket access:
  ```dockerfile
  RUN addgroup --system --gid 1001 nodejs
  RUN addgroup docker || true
  RUN adduser --system --uid 1001 nextjs
  RUN adduser nextjs docker || true
  ```
- Add environment variables for state.json and openclaw.json paths:
  ```dockerfile
  ENV STATE_FILE=/app/data/workspace-state.json
  ENV OPENCLAW_CONFIG=/app/data/openclaw.json
  ENV DOCKER_SOCKET=/var/run/docker.sock
  ```
- Add volume mount comments documenting the required mounts:
  ```dockerfile
  # Required volumes:
  #   -v /path/to/.openclaw:/app/data:ro          (state.json, openclaw.json)
  #   -v /var/run/docker.sock:/var/run/docker.sock  (Docker socket for log streaming)
  ```
- Keep the existing EXPOSE 6987 and PORT/HOSTNAME env vars

**Verify the build**:
