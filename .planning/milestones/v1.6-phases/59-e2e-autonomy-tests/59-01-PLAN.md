# Plan 59-01: E2E Test Infrastructure & Happy Path

**Objective**: Create Docker-based E2E testing infrastructure and implement happy path test for autonomy lifecycle.

**Context**: Phase 59 addresses the TEST-02 requirement gap identified in the v1.6 milestone audit. We need end-to-end tests that validate the full L3 autonomy lifecycle in realistic containerized environments.

---

## Requirements

- pytest + testcontainers for Docker orchestration
- Mock LLM responses for deterministic testing
- Event capture and verification
- CI/CD integration ready

---

## Tasks

### Task 1: E2E Test Infrastructure

**What**: Create Docker Compose setup and test fixtures for autonomy E2E tests.

**Implementation**:
```python
# tests/e2e/conftest.py
import pytest
from testcontainers.compose import DockerCompose

@pytest.fixture
def autonomy_stack():
    """Spin up orchestrator + mock LLM for E2E tests."""
    compose = DockerCompose("tests/e2e/fixtures")
    compose.start()
    yield compose
    compose.stop()
```

```yaml
# tests/e2e/fixtures/docker-compose.yml
version: '3.8'
services:
  orchestrator:
    build: ../../../
    environment:
      - AUTONOMY_ENABLED=1
      - MOCK_LLM_URL=http://mock-llm:8080
  mock-llm:
    image: openclaw/mock-llm:latest
    volumes:
      - ./responses:/responses
```

**Exit Criteria**:
- `tests/e2e/` directory created
- Docker Compose fixture spins up in <30 seconds
- Health check endpoint returns 200

### Task 4: CI Integration

**What**: Add E2E tests to GitHub Actions workflow.

**Implementation**:
```yaml
# .github/workflows/e2e.yml
- name: Run E2E Tests
  run: |
    cd tests/e2e
    docker-compose up --abort-on-container-exit
```

**Exit Criteria**:
- E2E job runs on PRs
- Job completes in <5 minutes
- Failed tests block merge
