**Part A: Shared vs Isolated pool modes (POOL-02)**

1. Add a `_shared_pool` module-level attribute to PoolRegistry for the global shared semaphore:
   ```python
   self._shared_semaphore: Optional[asyncio.Semaphore] = None
   self._shared_max: int = _POOL_DEFAULTS["max_concurrent"]
   ```

2. In `PoolRegistry.get_pool()`, after reading pool config (from Plan 01):
   - If `cfg["pool_mode"] == "isolated"`: pool gets its own semaphore (current behavior — dedicated per project_id)
   - If `cfg["pool_mode"] == "shared"`: pool shares `self._shared_semaphore`
     - Create the shared semaphore lazily on first shared-mode pool request
     - All shared-mode pools reference the same semaphore instance
     - When assigning shared semaphore to pool: `pool.semaphore = self._shared_semaphore`
   - If pool_mode changes between spawns (e.g., shared→isolated): update the pool's semaphore reference on next get_pool() call. Log at INFO.

3. Store pool_mode on the pool instance: `pool._pool_mode = cfg["pool_mode"]`

**Part B: Overflow policies (POOL-03)**

4. Modify `L3ContainerPool.spawn_and_monitor()` to implement overflow policy logic BEFORE the `async with self.semaphore:` block:

   - Read overflow policy from `self._pool_config` (attached by PoolRegistry in Plan 01)
   - **"reject" policy**: Check `self.semaphore._value == 0` before acquiring. If all slots occupied, raise a `PoolOverflowError` immediately with details:
     ```python
     class PoolOverflowError(Exception):
         """Raised when overflow_policy is 'reject' and all pool slots are occupied."""
         pass
     ```
     Include in message: slot count, running task IDs, suggestion to retry later.

   - **"wait" policy** (default): Use `asyncio.wait_for(self.semaphore.acquire(), timeout=queue_timeout_s)` instead of `async with self.semaphore`. If timeout expires, raise `PoolOverflowError` with timeout details. On successful acquire, use try/finally to release semaphore (replicate `async with` behavior).

   - **"priority" policy**: Implement a simple priority mechanism:
     - Add a `_priority_queue: asyncio.PriorityQueue` to L3ContainerPool
     - Priority spawns get lower priority number (higher priority): e.g., priority=0 vs standard=1
     - Add an optional `priority: int = 1` parameter to `spawn_and_monitor()`
     - When queue is used, tasks dequeue in priority order before acquiring semaphore
     - For the initial implementation, all spawns use default priority=1 unless caller specifies otherwise. The infrastructure is in place for L2 to pass priority=0 for elevated tasks.

5. Log overflow events at WARNING with project_id, task_id, policy, and queue state.

**Part C: Hot-reload for pool_mode and overflow_policy**

6. In `PoolRegistry.get_pool()`, compare current `pool._pool_config` with fresh config. If pool_mode or overflow_policy changed, update the pool instance:
   - Update `pool._pool_config` with fresh values
   - If pool_mode changed: swap semaphore reference (shared ↔ isolated)
   - overflow_policy change is automatic since it's read from `_pool_config` on each spawn
