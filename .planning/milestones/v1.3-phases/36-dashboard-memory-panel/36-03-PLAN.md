  <action>
1. Update `workspace/occc/src/components/memory/MemoryPanel.tsx` to wire search:
   - Add `searchQuery` state: `useState<string | null>(null)`
   - Pass `searchQuery` to `useMemory(projectId, searchQuery)` — SWR key changes trigger re-fetch with search param
   - Replace the search placeholder div with `<MemorySearch onSearch={q => setSearchQuery(q)} onClear={() => setSearchQuery(null)} isSearchMode={!!searchQuery} searchQuery={searchQuery ?? ''} />`
   - Reset `searchQuery` to null when project changes (useEffect on projectId)
   - Reset page to 1 when searchQuery changes

2. Wire single-item delete:
   - Add `deleteTarget` state: `useState<string | null>(null)` — ID of item pending deletion
   - On delete button click in MemoryRow (via callback prop): set `deleteTarget` to item ID, open ConfirmDialog
   - On confirm: call `fetch(\`/api/memory/${deleteTarget}\`, { method: 'DELETE' })`, then:
     a. On success: optimistically remove item from SWR cache via `mutate(prev => ({ ...prev, items: prev.items.filter(i => i.id !== deleteTarget) }), false)`
     b. Show `toast.success('Memory item deleted')` (import from react-toastify)
     c. On error: show `toast.error('Failed to delete memory item')`
   - Close dialog and reset `deleteTarget` to null

3. Wire bulk delete:
   - Add `selectedIds` state: `useState<Set<string>>(new Set())`
   - Track checkbox selections in MemoryTable via `onToggleSelect` and `onSelectAll` callbacks
   - When `selectedIds.size > 0`: show a floating "Delete selected ({n})" button above the table (between filters and table)
   - On bulk delete click: open ConfirmDialog with message "Delete {n} memory items? This cannot be undone."
   - On confirm: fire `Promise.all(selectedIds.map(id => fetch(\`/api/memory/${id}\`, { method: 'DELETE' })))`, then:
     a. Optimistically remove all selected from SWR cache via `mutate()`
     b. Show `toast.success(\`Deleted ${selectedIds.size} memory items\`)`
     c. Clear `selectedIds`
   - On error (any Promise rejects): show `toast.error('Some items could not be deleted')`, then do a full `mutate()` (refetch) to sync state

4. Add row removal animation:
   - On items that were just deleted: briefly apply `opacity-0 transition-opacity duration-300` before removing from DOM
   - Use a `deletingIds` state: when delete is confirmed, add ID(s) to `deletingIds`, wait 300ms, then remove from SWR cache
   - The fade-out gives visual feedback before the row disappears

CRITICAL: Import `toast` from `react-toastify` in MemoryPanel. The ToastContainer is already mounted globally in layout.tsx (from Plan 01).
