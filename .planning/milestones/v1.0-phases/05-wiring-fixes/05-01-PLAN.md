<task type="auto">
  <name>Task 2: Create automated delegation verification script</name>
  <files>scripts/verify_l1_delegation.py</files>
  <action>
Create `scripts/verify_l1_delegation.py` — an automated script that tests the L1->L2 delegation flow and reports pass/fail.

**The script must verify these stages:**

1. **Config Loading** — Load and validate `agents/clawdia_prime/agent/config.json`:
   - File exists and is valid JSON
   - Has `skill_registry` with `router` entry
   - `skill_path` points to existing directory (`skills/router_skill/`)
   - `skill.json` exists in the skill directory and is valid JSON

2. **Gateway Connectivity** — Check gateway is reachable at the configured endpoint:
   - Read gateway port from `openclaw.json` (port 18789)
   - Attempt TCP connection to `localhost:18789`
   - Report clear error if gateway is down: "Gateway not reachable at localhost:18789 — is the OpenClaw daemon running?"

3. **Delegation Roundtrip** — Attempt an actual delegation via router_skill:
   - Construct the dispatch command from skill.json handler template
   - Execute `node skills/router_skill/index.js pumplai_pm "ping"` in a subprocess
   - Capture stdout/stderr
   - If command returns non-zero: report the error clearly (gateway down, agent not found, timeout, etc.)
   - If command succeeds: parse JSON response, verify `status === 'ok'`

4. **Error Handling** — Each stage reports pass/fail independently:
   - Stage passes: green checkmark, brief message
   - Stage fails: red X, detailed error message with remediation hint
   - Script exits 0 if all stages pass, exits 1 if any stage fails
   - Use ANSI colors for terminal output (match monitor.py style)

**Important:** The script should handle the case where gateway or L2 agent is not running gracefully — it tests the WIRING, not the runtime availability. If the config is correct but gateway is down, report "WIRING: OK, RUNTIME: gateway not available" so the user knows the config is correct.

**Do NOT use external dependencies** — only Python stdlib (json, subprocess, socket, pathlib, sys).
