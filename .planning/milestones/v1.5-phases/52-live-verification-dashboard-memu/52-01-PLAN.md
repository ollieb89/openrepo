**Fix 1 — Health settings localStorage persistence (Test 5):**

In the `healthSettings` useState initializer, replace:
```typescript
const [healthSettings, setHealthSettings] = useState<HealthSettings>(DEFAULT_HEALTH_SETTINGS);
```
With a lazy initializer that reads from localStorage (same pattern as ThemeContext.tsx `occc-theme`):
```typescript
const [healthSettings, setHealthSettings] = useState<HealthSettings>(() => {
  if (typeof window === 'undefined') return DEFAULT_HEALTH_SETTINGS;
  try {
    const stored = localStorage.getItem('occc-health-settings');
    if (stored) return { ...DEFAULT_HEALTH_SETTINGS, ...JSON.parse(stored) };
  } catch {
    // Ignore parse errors
  }
  return DEFAULT_HEALTH_SETTINGS;
});
```

In the settings update handler (the function that receives updated settings from SettingsPanel's `onUpdate` prop), add localStorage write:
```typescript
// After setHealthSettings(settings):
try {
  localStorage.setItem('occc-health-settings', JSON.stringify(settings));
} catch {
  // Ignore quota errors
}
```

**Fix 2 — Auto-rescan after memory edit (Test 4):**

In `handleEditMemory` (the function that PUTs updated memory content), add `await runHealthScan()` AFTER the `mutate()` call. This ensures health flags reflect the updated content immediately.

Similarly, if `handleDeleteMemoryFromConflict` exists and dismisses a flag without re-scanning, add `await runHealthScan()` there too.

**Important:** Do NOT change any other behavior in MemoryPanel.tsx. These are surgical fixes.
