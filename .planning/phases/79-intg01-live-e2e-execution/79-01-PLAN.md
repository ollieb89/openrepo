  <action>
Run the following health checks in sequence. Each must pass before proceeding to the next. On any failure, STOP and report the exact service that failed with its remediation command — do not attempt the remaining checks.

**Check 1 — Docker:**
```bash
docker ps
```
Expected: exit 0 with any output (even empty table). If it fails: "Docker daemon not running. Start Docker Desktop or `sudo systemctl start docker`."

**Check 2 — memU service:**
```bash
make memory-health
```
Working directory: repo root. Expected: output contains "healthy" or HTTP 200 response. If it fails: "memU not running. Run `make memory-up` to start the Docker Compose service. Then re-run this plan."

If memU is not running, start it and verify:
```bash
make memory-up
sleep 5
make memory-health
```

**Check 3 — Gateway:**
```bash
curl -sf http://localhost:18789/health -o /dev/null -w "%{http_code}"
```
Expected: `200`. If it fails (connection refused or non-200):
- The gateway is started by running the openclaw Node.js runtime: `cd openclaw && pnpm run gateway:dev`
- This command must be run in the background (separate terminal or background process) before this plan executes
- Surface this instruction: "Gateway not running at :18789. In a separate terminal, run: `cd openclaw && pnpm run gateway:dev` — then re-run this plan."
- STOP — do not proceed to dashboard check until gateway is healthy.

**Check 4 — Dashboard:**
```bash
curl -sf http://localhost:6987 -o /dev/null -w "%{http_code}"
```
Expected: `200`. If it fails:
- Start dashboard: `make dashboard` (runs in foreground; must be started in a separate terminal)
- Surface: "Dashboard not running at :6987. In a separate terminal, run: `make dashboard` — then re-run this plan."
- STOP.

All 4 checks must pass before Task 2 runs.
