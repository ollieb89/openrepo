  <files>orchestration/snapshot.py</files>
  <action>
Add `_memorize_review_decision()` as a module-level helper in `orchestration/snapshot.py`. This is a fire-and-forget function that launches a daemon thread to POST the review decision to memU.

**Helper function `_memorize_review_decision()`:**
- Parameters: `project_id: str, task_id: str, verdict: str, reasoning: str, diff_summary: str = "", skill_type: str = ""`
- Wrap entire body in try/except that logs warning and returns silently (never raises)
- Import `get_memu_config` from `orchestration.project_config` and `AgentType` from `orchestration.memory_client` inside the function (lazy import, consistent with pool.py pattern)
- Read `memu_api_url` via `get_memu_config()`. If empty or project_id is falsy, log debug and return early
- Build content string with header lines: `# L2 Review Decision: task {task_id}`, `Verdict: {verdict}`, `Task type: {skill_type}`, `Reasoning: {reasoning}`, optional `Diff summary:\n{diff_summary[:500]}` section
- Define inner `_post()` function that uses `httpx.Client` (sync, same pattern as `_retrieve_memories_sync` in spawn.py) with `httpx.Timeout(10.0, connect=2.0)` to POST to `/memorize`
- Payload shape: `{"resource_url": content, "modality": "conversation", "user": {"user_id": project_id, "agent_type": AgentType.L2_PM.value}}`
- Inner `_post()` catches all exceptions, logs warning with task_id, verdict, error
- Launch thread: `threading.Thread(target=_post, daemon=True, name=f"memu-review-{task_id}")`
- Add `import threading` to module imports

**Wire into `l2_merge_staging()`:**
- Add new keyword parameters: `reasoning: str = "", skill_type: str = "", project_id: Optional[str] = None` — all with safe defaults to preserve backward compatibility
- On **success path** (after branch deletion, before the final `return`): call `_memorize_review_decision(project_id=project_id or "", task_id=task_id, verdict="merge", reasoning=reasoning, diff_summary="", skill_type=skill_type)`
- On **conflict-abort path** (after `merge --abort`, before return): call `_memorize_review_decision(project_id=project_id or "", task_id=task_id, verdict="conflict", reasoning=reasoning or f"Merge conflict in task {task_id}", diff_summary=merge_result.stderr[:500], skill_type=skill_type)`
- Do NOT call on `GitOperationError` paths — those are programming errors, not review decisions

**Wire into `l2_reject_staging()`:**
- Add new keyword parameters: `reasoning: str = "", skill_type: str = "", project_id: Optional[str] = None`
- After the branch force-delete succeeds (before state update block): call `_memorize_review_decision(project_id=project_id or "", task_id=task_id, verdict="reject", reasoning=reasoning, diff_summary="", skill_type=skill_type)`

**Update function docstrings** to document the new parameters and memorization behavior.
