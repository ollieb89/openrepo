    **Step 3: Create memory.py** with two classes:

    `MemoryProfiler`:
    - `__init__(self, project_id: str, decay_lambda: float = 0.05, exploration_rate: float = 0.20, min_threshold: int = 5)` — stores config, loads topology config defaults if not overridden
    - `_decay_weight(self, timestamp_iso: str) -> float` — returns `math.exp(-self.decay_lambda * age_days)`. Unknown/invalid timestamps return 1.0
    - `compute_profile(self) -> dict` — loads changelog via `load_changelog(self.project_id)`, iterates entries, for each entry: extract correction_type and determine approved archetype via `ArchetypeClassifier.classify()` on the approved topology (re-classify from current.json at that point, OR use annotations.approved_archetype if present). Apply signal mapping: initial=+1.0, soft=+0.5, hard=+1.0 to approved/-0.5 to proposed. Weight each signal by `_decay_weight(entry["timestamp"])`. Normalize affinities to [0, 10] scale. If correction_count < min_threshold, return default profile with threshold_status="below_threshold". Write profile to storage via `save_memory_profile()`. Return the profile dict.
    - `get_preference_fit(self, archetype: str, explore: bool = False) -> int` — loads profile from storage. If threshold_status != "active" OR explore==True, return 5. Otherwise return clamped round of archetype_affinity[archetype]. The `explore` param is determined by caller drawing a SINGLE random number per scoring session.
    - `get_report(self, detail: bool = False) -> dict` — returns dict with project_id, correction_count (total, soft, hard), threshold_status, archetype_affinity, top 3 patterns (from load_patterns). If detail=True, adds decay-weighted timeline and full pattern list with confidence scores.

    `PatternExtractor`:
    - `__init__(self, project_id: str, min_threshold: int = 5)` — stores project_id and threshold
    - `extract(self) -> list` — loads changelog. If len < min_threshold, returns []. Otherwise: loads existing patterns via load_patterns(). Builds LLM prompt with correction diffs and existing patterns (for deduplication). Calls `asyncio.run(call_llm(SYSTEM_PROMPT, user_message))`. Parses JSON response via `strip_markdown_fences()` + `json.loads()`. Discards patterns with confidence < 0.4. If total patterns > 20, prunes to top 10 by confidence. Saves via `save_patterns()`. Returns pattern list.
    - On LLM failure: log warning, return existing patterns unchanged. Never raise.

    LLM prompt constants (module-level):
    - `PATTERN_EXTRACTION_SYSTEM` — instructs LLM to analyze correction diffs and return JSON array of pattern objects with fields: pattern, confidence, source_correction_ids, archetype_bias
    - `PATTERN_EXTRACTION_USER` — template with placeholders for n, project_id, existing_patterns, correction_diffs
