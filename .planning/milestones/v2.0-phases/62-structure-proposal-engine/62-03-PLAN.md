**2. Create `rubric.py`:**

Following the research document pattern. 6 scored dimensions + 1 derived:

```python
DIMENSIONS = ["complexity", "coordination_overhead", "risk_containment",
              "time_to_first_output", "cost_estimate", "preference_fit"]

DEFAULT_WEIGHTS = {
    "complexity": 0.15, "coordination_overhead": 0.15,
    "risk_containment": 0.20, "time_to_first_output": 0.20,
    "cost_estimate": 0.10, "preference_fit": 0.20,
}

@dataclass
class RubricScore:
    complexity: int           # 0-10
    coordination_overhead: int
    risk_containment: int
    time_to_first_output: int
    cost_estimate: int
    preference_fit: int       # Always 5 pre-Phase 64
    overall_confidence: int   # Weighted average
    key_differentiators: List[str] = field(default_factory=list)
```

Score computation rules (all 0-10 integers):
- `complexity`: `max(0, 10 - node_count - edge_count // 2)` clamped to 0-10
- `coordination_overhead`: `max(0, 10 - coordination_edge_count * 3)` clamped
- `risk_containment`: `min(10, review_gate_count * 3 + escalation_count * 2)` clamped
- `time_to_first_output`: `max(0, 10 - max_chain_depth * 2)` clamped
- `cost_estimate`: `max(0, 10 - node_count * 2)` clamped
- `preference_fit`: always 5
- `overall_confidence`: `round(sum(score * weight for each dimension))` clamped 0-10

`find_key_differentiators(scores: list[RubricScore]) -> list[str]`:
- For each dimension, compute max - min across all scores
- Return dimension names where spread >= 3 (sorted by spread descending)

Create `packages/orchestration/src/openclaw/topology/linter.py`:

```python
MAX_RETRIES = 2  # 3 total attempts

@dataclass
class LintResult:
    valid: bool
    adjusted: bool
    rejected_roles: List[str]
    adjustments: List[str]
    proposal: dict  # The (possibly adjusted) proposal dict

class ConstraintLinter:
    def __init__(self, registry: AgentRegistry, max_concurrent: int):
        self.registry = registry
        self.max_concurrent = max_concurrent
        self._known_roles: set[str] = set(registry._agents.keys())

    def lint(self, archetype: str, proposal_data: dict) -> LintResult:
        """Validate and auto-adjust a single archetype proposal."""
        # Step 1: Check all role IDs against registry
        roles_in_proposal = [r["id"] for r in proposal_data.get("roles", [])]
        unknown = [r for r in roles_in_proposal if r not in self._known_roles]
        if unknown:
            return LintResult(valid=False, adjusted=False,
                            rejected_roles=unknown, adjustments=[], proposal=proposal_data)

        # Step 2: Check pool size (count of level-3 parallel roles)
        l3_roles = [r for r in proposal_data.get("roles", []) if r.get("level") == 3]
        if len(l3_roles) > self.max_concurrent:
            # Auto-constrain: reduce parallel L3 count
            # Prefer removing coordination-only roles over review-gate-connected roles
            adjusted_proposal, adj_log = self._auto_constrain(proposal_data, l3_roles)
            return LintResult(valid=True, adjusted=True,
                            rejected_roles=[], adjustments=adj_log, proposal=adjusted_proposal)

        return LintResult(valid=True, adjusted=False,
                        rejected_roles=[], adjustments=[], proposal=proposal_data)
```

`_auto_constrain(proposal_data, l3_roles)`:
- Count how many L3 roles to remove: `excess = len(l3_roles) - self.max_concurrent`
- Score each L3 role by "removal cost": roles connected to review_gate edges have high cost, roles with only coordination edges have low cost
- Remove lowest-cost roles first
- For each removed role, also remove edges referencing it
- If removing would eliminate ALL review-gate edges, add WARNING to adjustments
- Log each adjustment: f"Adjusted: {len(l3_roles)} parallel L3 → {self.max_concurrent} (max_concurrent limit)"
