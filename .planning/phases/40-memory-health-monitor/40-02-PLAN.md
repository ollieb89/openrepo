**HealthTab.tsx** — new component `workspace/occc/src/components/memory/HealthTab.tsx`:

Define TypeScript types at top of file:
```typescript
interface HealthFlag {
  memory_id: string;
  flag_type: 'stale' | 'conflict';
  score: number;
  recommendation: string;
  conflict_with?: string;
}
```

Component receives props: `flags: Map<string, HealthFlag>`, `onRunScan: () => void`, `scanRunning: boolean`, `onOpenConflict: (flag: HealthFlag) => void`, `onDismissFlag: (memoryId: string) => void`, `onOpenSettings: () => void`.

Layout:
1. **Summary bar** at top — colored count chips: "N stale" in orange pill, "N conflicts" in red pill. "Run Scan" button (disabled + spinner when scanRunning). Gear icon button to open settings (calls `onOpenSettings`).
2. **Flag list** below — sorted by score descending. Each row shows memory_id (truncated), flag type badge (orange/red pill same as list badges), score, recommendation text, and action button. Conflict flags get "View Conflict" button (calls `onOpenConflict`). Stale flags get "Dismiss" button (calls `onDismissFlag`).
3. Empty state when no flags: "No health issues found. Run a scan to check."

Use Tailwind classes consistent with existing components. Dark mode support using `dark:` variants matching MemoryRow patterns.

**MemoryPanel.tsx** — add health state management:

Add state:
- `healthFlags: Map<string, HealthFlag>` — keyed by memory_id
- `showOnlyFlagged: boolean` — filter toggle
- `activeTab: 'list' | 'health'` — tab state
- `scanRunning: boolean`
- `healthSettings` — object with `scan_interval_ms: number` (default 3600000), `age_threshold_days: number` (default 30), `retrieval_window_days: number` (default 14), `similarity_min: number` (default 0.75), `similarity_max: number` (default 0.97)

Add `runHealthScan()` async function:
- Sets `scanRunning = true`
- POSTs to `/api/memory/health-scan` with `{ user_id: projectId, ...healthSettings }`
- Parses response, builds Map from `data.flags`
- Sets `healthFlags` state
- Sets `scanRunning = false`

Add scheduled scan `useEffect`:
- When `activeTab === 'health'` and `healthSettings.scan_interval_ms > 0`, set up `setInterval` calling `runHealthScan()`
- Return cleanup function with `clearInterval` (Pitfall 5)
- Disable interval when `projectId` is null

Add tab bar above the existing content:
- Two tabs: "Memories" (list icon) and "Health" (with count badge showing total unresolved flags — red/orange number)
- Tab styling: bottom border on active tab, matching existing dashboard tab patterns

When `activeTab === 'list'`:
- Render existing memory list with a "Show only flagged" toggle button when healthFlags.size > 0
- When toggle active, filter memories to only those with keys in healthFlags
- Pass `healthFlag` prop to MemoryTable → MemoryRow for badge rendering

When `activeTab === 'health'`:
- Render HealthTab component with flags, scan handlers, settings handler

**MemoryRow.tsx** — add health badge:

Accept new optional prop `healthFlag?: HealthFlag`.

After existing category/type badges, render health badge conditionally:
- Stale: `bg-orange-100 text-orange-700 dark:bg-orange-900/40 dark:text-orange-300` pill with text "stale"
- Conflict: `bg-red-100 text-red-700 dark:bg-red-900/40 dark:text-red-300` pill with text "conflict"
- Badge is clickable — `onClick` stops propagation and (for conflicts) triggers conflict panel open

Update **MemoryTable.tsx** to pass `healthFlag` from a `healthFlags` Map prop down to each MemoryRow (lookup by `memory.id`).
