<objective>
Add in-memory state caching to JarvisState with mtime-based invalidation and write-through semantics, so that monitor polling and dashboard reads serve from memory without competing with spawn writes.

Purpose: Reduce disk I/O for the hot read path (monitor polls every 1s, dashboard polls every 3-5s). Writes remain exclusive-locked and atomic, but the cache eliminates redundant disk reads between writes. This directly addresses PERF-02 (caching), PERF-03 (write-through avoids re-read after write), and PERF-04 (reads use shared locks only when cache misses).
Output: Modified `state_engine.py` with per-instance cache and `config.py` with cache TTL constant.
</objective>

**In `orchestration/state_engine.py`:**

1. Add `os` import (for `os.path.getmtime`) and import `CACHE_TTL_SECONDS` from config:
   ```python

   from .config import LOCK_TIMEOUT, LOCK_RETRY_ATTEMPTS, CACHE_TTL_SECONDS
   ```

2. Add per-instance cache fields in `__init__`:
   ```python
   self._cache: Optional[Dict[str, Any]] = None
   self._cache_mtime: float = 0.0
   self._cache_time: float = 0.0  # time.time() when cache was populated
   ```

3. Add a private method `_is_cache_valid() -> bool`:
   - If `_cache` is None, return False (cache miss — first read)
   - If `time.time() - _cache_time > CACHE_TTL_SECONDS`, return False (TTL expired — safety net)
   - Try `os.path.getmtime(self.state_file)` — if it differs from `_cache_mtime`, return False (external modification)
   - If getmtime raises OSError (file deleted), return False
   - Otherwise return True (cache hit)

4. Modify `read_state()`:
   - Before acquiring any lock, call `_is_cache_valid()`
   - On cache hit: log at DEBUG "State cache hit" with extra={"state_file": str(self.state_file)}, return `copy.deepcopy(self._cache)` (deep copy prevents caller mutation of cache)
   - On cache miss: log at DEBUG "State cache miss" with extra={"reason": "..."} where reason is "first_read", "ttl_expired", "mtime_changed", or "file_missing"
   - After disk read (existing path with LOCK_SH), update cache: `self._cache = copy.deepcopy(state)`, `self._cache_mtime = os.path.getmtime(self.state_file)`, `self._cache_time = time.time()`
   - Return the read state as before

5. Add `import copy` at the top.

6. read_state() continues to use LOCK_SH for disk reads — shared locks only, per PERF-04. No changes to lock type.

**Do NOT change `read_task()`** — it delegates to `read_state()` which now uses the cache.

**Do NOT change lock semantics** — writes still use LOCK_EX, reads still use LOCK_SH. The cache simply avoids hitting disk when the file hasn't changed.
