Create `skills/notion-kanban-sync/notion_client.py` — the HTTP wrapper for all Notion API interaction.

**Class: NotionClient**

Constructor `__init__(self, config_path: str = None)`:
- Reads `NOTION_TOKEN` from environment. Raises `RuntimeError("NOTION_TOKEN not set")` if missing.
- Sets headers: `Authorization: Bearer {token}`, `Notion-Version: 2025-09-03`, `Content-Type: application/json`.
- Loads config.json (default path: `Path(__file__).parent / "config.json"`).
- Module-level `threading.Lock()` for config reads/writes (per Research pitfall 4).

**Core HTTP method `_request(self, method, path, **kwargs) -> dict`:**
- Uses httpx to make requests to `https://api.notion.com{path}`.
- Retry logic: on 429 → sleep `base_delay * 2^attempt`, retry up to `retry_max_attempts`. On 5xx → sleep 2s, retry once. On 400 → do NOT retry, raise. On 404 → raise (caller handles cache invalidation).
- Rate limiting: `time.sleep(0.35)` between requests when `self._bulk_mode` is True (for reconcile). Default bulk_mode=False.
- Returns `resp.json()`.

**Typed helpers:**
- `search_database(self, name: str, signature_property: str) -> Optional[dict]` — POST `/v1/search` with `query=name, filter={"value": "database", "property": "object"}`. Filter results for `object == "data_source"` with title matching `name` AND properties containing `signature_property`. Returns `{"database_id": ..., "data_source_id": ...}` or None.
- `create_database(self, parent_page_id: str, title: str, properties: dict) -> dict` — POST `/v1/data_sources` with parent, title, properties, is_inline=True. Returns `{"database_id": result["id"], "data_source_id": result.get("data_source_id", result["id"])}`.
- `query_database(self, data_source_id: str, filter_: dict) -> list` — POST `/v1/data_sources/{data_source_id}/query`. Uses data_source_id NOT database_id (API 2025-09-03 requirement).
- `create_page(self, database_id: str, properties: dict) -> dict` — POST `/v1/pages` with parent database_id.
- `update_page(self, page_id: str, properties: dict) -> dict` — PATCH `/v1/pages/{page_id}`.
- `get_page(self, page_id: str) -> dict` — GET `/v1/pages/{page_id}` (for read-before-write on Activity field).
- `upsert_by_dedupe(self, db_id: str, ds_id: str, dedupe_property: str, dedupe_value: str, properties: dict) -> dict` — search by dedupe property, create or update. Returns `{"action": "created"|"updated", "page_id": ..., "properties": ...}`.

**Bootstrap method `bootstrap(self) -> None`:**
1. If `notion_projects_db_id` is non-null in config → validate with a GET (if 404 → clear and re-discover).
2. Search for Projects DB: `search_database("Projects", "OpenClaw ID")`.
3. If not found and `auto_create_dbs` → create with Projects DB schema (from SPEC.md: Name title, OpenClaw ID rich_text, Type select, Status select, Repo/Path url, Current Phase rich_text, Milestone rich_text, Sync Status rich_text, Sync Error rich_text, Notes rich_text, Priority select).
4. If not found and not auto_create → raise `RuntimeError("Projects DB not found. Set notion_projects_db_id in config.json or enable auto_create_dbs")`.
5. If auto_create and `notion_parent_page_id` is null → raise `RuntimeError("Set notion_parent_page_id in config.json to enable auto-creation")`.
6. Repeat for Cards DB: search for "Cards" with "OpenClaw Phase ID" signature property. Schema from SPEC.md (all 15 properties: Name title, Status select, Area select, Project relation, Card Type select, Capture Source select, OpenClaw Phase ID rich_text, OpenClaw Event Anchor rich_text, Capture Hash rich_text, Priority select, Target Week date, Notes rich_text, Activity rich_text, Last Activity At date, Last Synced date).
7. Cache all 4 IDs (projects_db_id, projects_ds_id, cards_db_id, cards_ds_id) to config.json under Lock.

**Config persistence:**
- `_load_config(self) -> dict` — reads config.json under Lock.
- `_save_config(self, config: dict) -> None` — writes config.json under Lock with indent=2.
- `_get_db_ids(self) -> tuple` — returns (projects_db_id, projects_ds_id, cards_db_id, cards_ds_id) from config, calling bootstrap() if any are None.

**Activity append helper:**
- `append_activity(self, page_id: str, new_line: str) -> dict` — GET page to read current Activity text, prepend `[{timestamp}] {new_line}\n` to existing content, PATCH with full combined text. Also update Last Activity At and Last Synced. This is 2 API calls per append (per Research pitfall 7).
