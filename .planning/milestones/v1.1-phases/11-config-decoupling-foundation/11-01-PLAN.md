Add a module-level helper function `_detect_default_branch(workspace: Path, project_id: Optional[str] = None) -> str` to `snapshot.py`. Place it near the top of the file after imports.

**Resolution order** (per locked decisions):
1. Check `default_branch` field in project.json via `load_project_config(project_id)`. If field exists and is non-empty, return it.
2. Run `git -C <workspace> symbolic-ref refs/remotes/origin/HEAD`, parse the branch name from the output (split on `/`, take last element). Return if successful.
3. Check if `main` branch exists locally: `git -C <workspace> rev-parse --verify main`
4. Check if `master` branch exists locally: `git -C <workspace> rev-parse --verify master`
5. Last resort fallback: return `"main"` (Claude's discretion — preserves current behavior). Log a warning via `print(f"WARNING: Could not detect default branch for {workspace}, falling back to 'main'")`.

**Per locked decision**: Fresh detection on every call — do NOT cache results.

Add import for `load_project_config` from `.project_config` (or `orchestration.project_config` — match the existing import style in snapshot.py). If snapshot.py already imports from project_config, add to that import.

This task ONLY adds the helper function. It does NOT modify any existing functions in snapshot.py — that happens in Plan 02.
