  <behavior>
    - CorrectionSession: cycle_count starts at 0, cycle_limit_reached is True when cycle_count >= max_cycles (3)
    - apply_soft_correction: increments cycle_count, calls generate_proposals_sync with feedback in clarifications["user_feedback"], returns ProposalSet; updates best_proposal_set if new set's top proposal has higher overall_confidence
    - export_draft: writes proposal-draft.json with __comment__nodes and __comment__edges keys; file is valid JSON; path is under topology dir
    - import_draft: loads proposal-draft.json, strips keys starting with "__comment__", calls TopologyGraph.from_dict; returns (TopologyGraph, LintResult); raises clear error for invalid edge_type values
    - approve_topology: calls load_topology to get previous, computes topology_diff if previous exists, calls save_topology then append_changelog; changelog entry has timestamp, correction_type, diff (via .to_dict()), and annotations
    - compute_pushback_note: returns empty string when original overall_confidence < pushback_threshold; returns dimension-specific note when confidence >= threshold and any dimension drops by >= 2; never raises
    - check_approval_gate: returns {"approved": False, "reason": "..."} when current.json absent and auto_approve_l1 is False; returns {"approved": True} when current.json exists or auto_approve_l1 is True
    - save_pending_proposals: atomic write with fcntl LOCK_EX + tmp rename to pending-proposals.json
    - load_pending_proposals: returns None if file missing, otherwise loads with fcntl LOCK_SH
  </behavior>
  <action>
    **correction.py** — Create with:

    1. `CorrectionSession` dataclass: fields for outcome, project_id, proposal_set (ProposalSet), best_proposal_set (ProposalSet), cycle_count (int, default 0), max_cycles (int, default 3), approved_proposal (Optional[TopologyProposal], default None), correction_history (List[dict], default_factory=list), clarifications (dict, default_factory=dict), registry (any), max_concurrent (int, default 3). Property `cycle_limit_reached` returns `self.cycle_count >= self.max_cycles`.

    2. `apply_soft_correction(session: CorrectionSession, feedback: str, weights: dict) -> ProposalSet`:
       - Check cycle_limit_reached BEFORE LLM call — if True, raise ValueError("Cycle limit reached")
       - Increment cycle_count
       - Build augmented clarifications: `{**session.clarifications, "user_feedback": feedback}`
       - Call `generate_proposals_sync(outcome=session.outcome, project_id=session.project_id, registry=session.registry, max_concurrent=session.max_concurrent, clarifications=augmented)`
       - Lint each archetype with ConstraintLinter — same pattern as propose.py main()
       - Build proposals via `build_proposals()` from proposer, convert to proposal_models.TopologyProposal using the same `_to_pm_proposals` pattern from propose.py (extract that helper to a shared location or inline it)
       - Score each proposal with `score_proposal(p.topology, weights)`
       - Build ProposalSet, update session.proposal_set
       - If top proposal's overall_confidence > best_proposal_set's top confidence, update best_proposal_set
       - Record correction in session.correction_history: {"type": "soft", "feedback": feedback, "cycle": session.cycle_count}
       - Return the new ProposalSet

    3. `export_draft(proposal: TopologyProposal, project_id: str) -> Path`:
       - Get topo_dir from storage._topology_dir(project_id)
       - Build data = proposal.topology.to_dict()
       - Add `data["__comment__nodes"] = "Nodes: each has id (string), level (1/2/3), intent (what role does), risk_level (low/medium/high)"`
       - Add `data["__comment__edges"] = "Edges: from_role/to_role (node ids), edge_type: delegation | coordination | review_gate | escalation | information_flow"`
       - Write to `topo_dir / "proposal-draft.json"` using json.dump with indent=2
       - Return the path

    4. `import_draft(project_id: str, registry, max_concurrent: int) -> tuple[TopologyGraph, LintResult]`:
       - Load from `topo_dir / "proposal-draft.json"`
       - Strip keys starting with "__comment__"
       - Wrap `TopologyGraph.from_dict(clean)` in try/except ValueError — re-raise with message listing valid EdgeType values
       - Run ConstraintLinter.lint("hard_correction", proposal_dict) where proposal_dict has "roles" and "edges" keys matching linter expectations
       - Return (graph, lint_result)

    **approval.py** — Create with:

    1. `approve_topology(project_id, approved_graph, correction_type, pushback_note="") -> dict`:
       - Load previous topology via load_topology(project_id)
       - Compute diff via topology_diff(previous, approved_graph) if previous exists, else None
       - Build entry dict: timestamp (UTC ISO), correction_type, diff (via .to_dict() — CRITICAL for EdgeType serialization), annotations (with pushback_note if non-empty)
       - Call save_topology(project_id, approved_graph) then append_changelog(project_id, entry)
       - Delete pending-proposals.json if it exists (per research recommendation)
       - Return the entry dict

    2. `compute_pushback_note(original_score: RubricScore, approved_graph: TopologyGraph, weights: dict, pushback_threshold: int = 8) -> str`:
       - If original_score.overall_confidence < pushback_threshold: return ""
       - Score approved_graph via score_proposal(approved_graph, weights)
       - Compare each dimension (complexity, coordination_overhead, risk_containment, time_to_first_output, cost_estimate). If original - approved >= 2, add to dimension_notes list with label and values
       - If no dimension_notes: return ""
       - Return "Note: My original proposal scored higher on {joined notes}. This is informational only — it does not block execution."

    3. `check_approval_gate(project_id: str, auto_approve_l1: bool = False) -> dict`:
       - If auto_approve_l1: return {"approved": True}
       - Check if current.json exists via load_topology(project_id) is not None
       - If exists: return {"approved": True}
       - If not: return {"approved": False, "reason": f"No approved topology for project '{project_id}'. Run 'openclaw-propose' to generate and approve a topology."}

    **storage.py** — Extend with:

    1. `save_pending_proposals(project_id: str, data: dict) -> None`: same atomic tmp+rename+fcntl pattern as save_topology. File: `pending-proposals.json`.

    2. `load_pending_proposals(project_id: str) -> Optional[dict]`: returns None if file missing. Uses fcntl LOCK_SH for read.

    3. `delete_pending_proposals(project_id: str) -> None`: deletes pending-proposals.json if it exists.
