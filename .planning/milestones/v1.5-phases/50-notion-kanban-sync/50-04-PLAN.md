  <action>
Add container event handlers to notion_sync.py. These handle `container_completed` and `container_failed` events.

1. `_sync_container_completed(event: dict, result: SyncResult) -> None`:
   - Find parent phase card by `openclaw_phase_id` = `{project_id}:{phase_id}`.
   - If parent card not found → log warning, skip (orphan container event).
   - Evaluate **meaningful container rule** (deterministic, from SPEC):
     - `runtime_seconds = event["payload"].get("runtime_seconds", 0)`
     - `requires_human_review = event["payload"].get("requires_human_review", False)`
     - `failure_category = event["payload"].get("failure_category")`
     - `meaningful_runtime_threshold = config["meaningful_container_runtime_min"] * 60` (convert minutes to seconds)
     - `is_meaningful = (runtime_seconds > meaningful_runtime_threshold) or requires_human_review or (failure_category in ("tests_failed", "lint_failed", "deploy_failed"))`
   - **If meaningful**: Create a child card in Cards DB:
     - Name: "L3: {container_id} ({runtime_seconds}s)"
     - Status: "Done"
     - Area: "Dev"
     - Card Type: "Task"
     - Capture Source: "OpenClaw Event"
     - OpenClaw Event Anchor: `{project_id}:{container_id}:container_completed`
     - Project: relation to project page
     - Last Synced: now
   - **Always** (meaningful or not): Append to parent phase card Activity:
     - `"container_completed: {container_id} ({runtime_seconds}s)"`
     - Use `client.append_activity(parent_page_id, activity_line)`.

2. `_sync_container_failed(event: dict, result: SyncResult) -> None`:
   - Find parent phase card by `openclaw_phase_id`.
   - Evaluate meaningful rule (same as above — failures with actionable category create cards).
   - If meaningful: Create child card with Status="Waiting", Card Type="Bug" or "Incident".
   - Always append to parent phase card Activity: `"container_failed: {container_id} (exit {exit_code})"`.
   - **Retries exhausted rule**: If `payload.retry_count >= config.retry_max_attempts`:
     - Set parent phase card Status → "Waiting" (but ONLY if `_is_openclaw_linked(parent_card)`).
     - Append additional activity: `"retries exhausted — phase blocked"`.

3. `_evaluate_meaningful_rule(event: dict, config: dict) -> bool`:
   - Pure function encapsulating the 3-condition check.
   - Reads `meaningful_container_runtime_min` from config (default 10 minutes).
   - Returns True if any of the 3 conditions met.

  <name>Task 2: Enforce status ownership carve-out across all handlers</name>
  <files>

  <action>
Audit and enforce the status ownership carve-out across ALL event handlers in notion_sync.py.

The rule (from SPEC Decision #12):
- **OpenClaw-linked cards** (has non-empty `openclaw_phase_id` or `openclaw_event_anchor`): OpenClaw owns Status. Write Status on sync.
- **Conversation-captured cards** (has `capture_source = "Conversation"` but no OpenClaw linkage): Notion owns Status. Never write Status.
- **Manual cards** (has `capture_source = "Manual"` or no capture_source): Notion owns Status. Never write Status.

Implementation:

1. Refactor `_safe_set_status()` (from Plan 03) to handle all cases:
   ```python
   def _should_write_status(page: dict) -> bool:
       """Return True only if OpenClaw owns Status on this card."""
       props = page.get("properties", {})
       phase_id = _get_rich_text_value(props.get("OpenClaw Phase ID", {}))
       event_anchor = _get_rich_text_value(props.get("OpenClaw Event Anchor", {}))
       return bool(phase_id or event_anchor)
   ```

2. Add `_get_rich_text_value(prop: dict) -> str` helper — extracts plain text from Notion rich_text property format. Returns empty string if property is empty or missing.

3. Audit every handler that writes Status:
   - `_sync_phase_started` — writes "In Progress" → guard with `_should_write_status()` on update path (create path is always safe since we're creating an OpenClaw-linked card)
   - `_sync_phase_completed` — writes "Done" → guard
   - `_sync_phase_blocked` — writes "Waiting" → guard
   - `_sync_container_completed` — creates new card (always safe) + may update parent (guard)
   - `_sync_container_failed` — may update parent Status to "Waiting" (guard)
   - `_sync_project_removed` — writes "Archived" on Projects DB row → Projects DB rows are always OpenClaw-linked (safe)

4. For each guarded write, if `_should_write_status()` returns False:
   - Skip the Status field in the properties dict
   - Log: `"Skipping Status write on Notion-owned card {page_id}"`
   - Increment `result.skipped` count

5. Verify that `_is_openclaw_linked()` and `_should_write_status()` are used consistently (they should be the same function — consolidate if both exist from Plan 03).

CRITICAL: The create path (new cards from events) always sets Status because OpenClaw is creating an OpenClaw-linked card. The guard is only needed on the update path (existing cards found by dedupe key).
