Create `src/lib/redaction.ts` implementing server-side redaction (SEC-02). CRITICAL: All redaction happens server-side before data reaches the browser. Client-side redaction is security theater.

Implement `REDACTION_PATTERNS` array with these categories:
1. **AWS keys:** `AKIA[0-9A-Z]{16}` -> `[REDACTED_AWS_KEY]`
2. **OpenAI keys:** `sk-[a-zA-Z0-9]{20,}` -> `[REDACTED_API_KEY]`
3. **Anthropic keys:** `sk-ant-[a-zA-Z0-9-]{20,}` -> `[REDACTED_API_KEY]`
4. **Google keys:** `AIza[0-9A-Za-z_-]{35}` -> `[REDACTED_GOOGLE_KEY]`
5. **GitHub tokens:** `gh[ps]_[a-zA-Z0-9]{36,}` -> `[REDACTED_GITHUB_TOKEN]`
6. **Slack tokens:** `xox[pboa]-[0-9]+-[0-9A-Za-z-]+` -> `[REDACTED_SLACK_TOKEN]`
7. **Generic auth headers:** `authorization:\s*bearer\s+[^\s]+` (case-insensitive) -> `authorization: [REDACTED]`
8. **Generic API key headers:** `x-api-key:\s*[^\s]+` (case-insensitive) -> `x-api-key: [REDACTED]`
9. **Email addresses:** `\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b` -> `[REDACTED_EMAIL]`
10. **Generic secrets in env format:** `(?:PASSWORD|SECRET|TOKEN|API_KEY|PRIVATE_KEY)\s*[=:]\s*\S+` (case-insensitive) -> match key name + `=[REDACTED]`

Export:
- `redactSensitiveData(text: string): string` - applies all patterns sequentially
- `REDACTION_PATTERNS` array (exported for testing)
- `RedactionResult` type: `{ redacted: string, patternsMatched: string[] }`
- `redactWithReport(text: string): RedactionResult` - same as above but reports which patterns matched (for telemetry)

Create `src/lib/docker.ts` as Docker API client wrapper:
- Initialize dockerode with `{ socketPath: process.env.DOCKER_SOCKET || '/var/run/docker.sock' }`
- `getContainerByLabel(label: string, value: string): Promise<Docker.ContainerInfo | null>` - finds container by label filter (`docker ps --filter "label=key=value"`)
- `getContainerByName(name: string): Promise<Docker.ContainerInfo | null>` - finds container by name pattern
- `streamContainerLogs(containerId: string, onLog: (line: string) => void, signal: AbortSignal, options?: { tail?: number }): Promise<void>` - streams logs with `follow: true, stdout: true, stderr: true, timestamps: true, tail: options.tail || 100`
- Docker multiplexed stream handling: parse 8-byte header (stream type + length) from raw stream chunks to extract clean log lines
- Apply `redactSensitiveData()` to every log line BEFORE calling `onLog()`
- Handle stream errors and abort signal for cleanup
- `listSwarmContainers(): Promise<Docker.ContainerInfo[]>` - lists containers with label `openclaw.managed=true` or name pattern `openclaw-*`
- Graceful fallback: if Docker socket is not available, log warning and return empty results (don't crash the dashboard)

Create `src/app/api/logs/[agent]/route.ts` as SSE endpoint:
- Extract `agent` from route params (Next.js App Router dynamic route)
- Look up container ID for the agent using `getContainerByName()` or `getContainerByLabel('openclaw.agent_id', agent)` from lib/docker.ts
- If container not found, return 404 JSON: `{ error: 'Container not found for agent', agent }`
- Create ReadableStream that pipes dockerode log stream through redaction to SSE format
- Each log line sent as: `data: {"timestamp":"...","level":"info|warn|error|debug","message":"..."}\n\n`
- Parse log level from line content: lines containing "ERROR" or "error" -> "error", "WARN" or "warn" -> "warn", "DEBUG" or "debug" -> "debug", else "info"
- Parse timestamp from Docker timestamp prefix (ISO 8601 format at start of line)
- Send keepalive every 30s: `: keepalive\n\n`
- Set SSE headers (Content-Type: text/event-stream, Cache-Control: no-cache, Connection: keep-alive)
- Handle container stop/restart gracefully: send `event: container-stopped\ndata: {}\n\n` and close stream
- Accept optional query param `?tail=N` (default 100) for initial log buffer size

Create `src/hooks/useLogStream.ts` as client-side hook:
- `"use client"` directive
- `useLogStream(agentId: string | null)` hook
- When agentId is null, do nothing (no agent selected)
- When agentId changes: close existing EventSource, open new one to `/api/logs/${agentId}`
- Maintain `logs` state as array of `LogEntry` objects: `{ timestamp: string, level: 'debug' | 'info' | 'warn' | 'error', message: string }`
- Cap log buffer at 1000 entries (circular buffer: drop oldest when exceeding limit per research recommendation)
- On `message` event: parse JSON, append to logs
- On `container-stopped` event: append system message "Container stopped" with level "warn"
- On `error` event: set connection status to 'disconnected', attempt reconnect with exponential backoff (1s, 2s, 4s, max 30s)
- Export: `{ logs: LogEntry[], isConnected: boolean, clearLogs: () => void, connectionStatus: 'connected' | 'connecting' | 'disconnected' }`
- Clean up EventSource on unmount or agentId change
