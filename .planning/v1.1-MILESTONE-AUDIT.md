## Cross-Phase Integration

### Wiring Issues (3)

1. **monitor.py → project_config.py** (CFG-01): `get_state_path` imported but never called — `_discover_projects()` builds equivalent path inline. Functionally correct but API not consumed.

2. **snapshot.py → project_config.py** (CFG-02): `capture_semantic_snapshot()` and `cleanup_old_snapshots()` call `get_snapshot_dir()` with no `project_id` — resolves via ambient config, not explicit threading. Multi-project snapshot isolation depends on env var being set.

3. **snapshot.py create_staging_branch** (CFG-06): Has duplicate inline branch detection that skips project.json `default_branch` lookup. Other snapshot functions correctly use `_detect_default_branch()`.

### Broken Flows (3)

1. **Multi-project snapshot capture**: Snapshot directory resolves via ambient config, not explicit `project_id` parameter.

2. **Staging branch with custom default_branch**: `create_staging_branch()` ignores `default_branch` from project.json.

3. **Soul render for new projects**: Missing `agent_display_name` produces `# Soul: (L2)` — no error raised but semantically broken output.
