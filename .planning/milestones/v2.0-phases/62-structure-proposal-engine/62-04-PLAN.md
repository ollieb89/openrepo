A minimal async LLM client using httpx. Provider-agnostic — supports Anthropic and Google Gemini via environment variables. This is Claude's discretion per CONTEXT.md.

```python
import os
import re
import json
import httpx

DEFAULT_PROVIDER = "anthropic"  # fallback if not set

async def call_llm(system_prompt: str, user_message: str) -> str:
    """Call LLM and return raw text response. Provider from env vars."""
    provider = os.environ.get("OPENCLAW_LLM_PROVIDER", DEFAULT_PROVIDER)
    if provider == "anthropic":
        return await _call_anthropic(system_prompt, user_message)
    elif provider == "gemini":
        return await _call_gemini(system_prompt, user_message)
    else:
        raise ValueError(f"Unknown LLM provider: {provider}")

async def _call_anthropic(system_prompt: str, user_message: str) -> str:
    api_key = os.environ.get("ANTHROPIC_API_KEY") or os.environ.get("ANTHROPIC_TOKEN")
    if not api_key:
        raise RuntimeError("ANTHROPIC_API_KEY or ANTHROPIC_TOKEN env var required")
    async with httpx.AsyncClient(timeout=60.0) as client:
        resp = await client.post(
            "https://api.anthropic.com/v1/messages",
            headers={
                "x-api-key": api_key,
                "anthropic-version": "2023-06-01",
                "content-type": "application/json",
            },
            json={
                "model": os.environ.get("OPENCLAW_LLM_MODEL", "claude-sonnet-4-20250514"),
                "max_tokens": 4096,
                "system": system_prompt,
                "messages": [{"role": "user", "content": user_message}],
            },
        )
        resp.raise_for_status()
        data = resp.json()
        return data["content"][0]["text"]

async def _call_gemini(system_prompt: str, user_message: str) -> str:
    api_key = os.environ.get("GEMINI_API_KEY")
    if not api_key:
        raise RuntimeError("GEMINI_API_KEY env var required")
    model = os.environ.get("OPENCLAW_LLM_MODEL", "gemini-2.5-flash")
    async with httpx.AsyncClient(timeout=60.0) as client:
        resp = await client.post(
            f"https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent",
            params={"key": api_key},
            json={
                "system_instruction": {"parts": [{"text": system_prompt}]},
                "contents": [{"parts": [{"text": user_message}]}],
            },
        )
        resp.raise_for_status()
        data = resp.json()
        return data["candidates"][0]["content"]["parts"][0]["text"]

def strip_markdown_fences(text: str) -> str:
    """Strip ```json ... ``` fences from LLM response (pitfall 1 from research)."""
    return re.sub(r'^```(?:json)?\s*|\s*```$', '', text.strip())
```

Keep this minimal. Error handling: raise on HTTP errors, let caller handle retries. `strip_markdown_fences` is a utility used by the proposer.

2. `PROPOSAL_SYSTEM_PROMPT` — template string:
   ```
   You are an expert at designing multi-agent swarm topologies.
   Given an outcome description, generate exactly 3 topology proposals:
   one Lean (minimal roles, fast), one Balanced (moderate structure), one Robust (safe, redundant).

   Available agent roles: {available_roles}
   Project constraint: max {max_concurrent} concurrent L3 agents

   {rejection_context}

   Return ONLY valid JSON matching this schema: {json_schema}
   Do not include markdown fences or explanation outside the JSON.
   ```

3. `CLARIFYING_QUESTIONS = [("risk_tolerance", "What is your risk tolerance? (low/medium/high, default: medium)"), ("timeline_pressure", "Timeline pressure? (relaxed/moderate/urgent, default: moderate)")]`

   - If interactive: print each question, read input, default if empty
   - If not interactive: return defaults, note assumptions

5. `def _load_rejection_context(project_id: str) -> str | None`:
   - Read changelog via `load_changelog(project_id)`
   - Filter entries with `correction_type == "rejected"` or similar
   - Return formatted string for LLM or None if empty/missing
   - Gracefully degrade on missing/unexpected format (per research open question 3)

   - Call `call_llm(system_prompt, user_message)`
   - Strip markdown fences, parse JSON
   - Validate against PROPOSAL_JSON_SCHEMA using jsonschema
   - Return raw dict
