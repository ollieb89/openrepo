In `update_task()` (lines 381-404), the code correctly calls `get_active_project_id()` locally. Fix `create_task()` to match `update_task()` pattern: call `project_id = get_active_project_id()` before the event publishing block, then use `project_id=project_id` in the OrchestratorEvent.

    2. **`asyncio.run()` fails when called in the except block.** The pattern `asyncio.get_running_loop()` -> fails -> `asyncio.run(event_bridge.publish(...))` is fragile. The event bridge publish may itself have issues when called synchronously. Wrap the entire event publishing block in a try/except that catches ALL exceptions and logs a warning — event publishing should never crash core state operations. Pattern:
    ```python
    try:
        project_id = get_active_project_id()
        try:
            loop = asyncio.get_running_loop()
            loop.create_task(event_bridge.publish(...))
        except RuntimeError:
            asyncio.run(event_bridge.publish(...))
    except Exception as exc:
        logger.warning("Event publish failed (non-fatal)", extra={"error": str(exc)})
    ```

Add `@patch("openclaw.state_engine.event_bridge")` to the test decorator stack, with the mock accepting any publish call. The mock should make `publish` return a coroutine (use `AsyncMock` or make it return a completed future).
