   class SyncResult:
       def __init__(self, request_type):
           self.request_type = request_type

           self.skipped = 0
           self.errors = []
           self.mutations = []
       def record_mutation(self, action, target, page_id, dedupe_key): ...

**event_bus_hook.py — subscription registration:**

This file registers Notion sync handlers with the event bus. It must be imported before events fire.

```python
import os
import logging

logger = logging.getLogger("openclaw.notion_hook")

# Only register if NOTION_TOKEN is configured
if os.environ.get("NOTION_TOKEN"):
    try:
        from openclaw.event_bus import subscribe

        def _handle_event(envelope: dict) -> None:
            """Route event to notion_sync handler. Fire-and-forget."""
            import sys, os
            skill_dir = os.path.join(os.path.dirname(__file__))
            if skill_dir not in sys.path:
                sys.path.insert(0, skill_dir)
            from notion_sync import handle_event_sync
            handle_event_sync({"request_type": "event_sync", "event": envelope})

        for evt in ("phase_started", "phase_completed", "phase_blocked",
                     "container_completed", "container_failed",
                     "project_registered", "project_removed"):
            subscribe(evt, _handle_event)

        logger.info("Notion sync handlers registered for 7 event types")
    except ImportError:
        logger.debug("event_bus not available — Notion hooks not registered")
else:
    logger.debug("NOTION_TOKEN not set — Notion sync disabled")
```

Add phase lifecycle event handlers to notion_sync.py:

1. `_sync_phase_started(event: dict, result: SyncResult) -> None`:
   - Dedupe key: `openclaw_phase_id` = `{project_id}:{phase_id}`.
   - Upsert Cards DB:
     - Name: "Phase {phase_id}: {payload.phase_name}" (or just "Phase {phase_id}" if phase_name missing)
     - Status: "In Progress"
     - Area: "Dev"
     - Card Type: "Phase"
     - Capture Source: "OpenClaw Event"
     - OpenClaw Phase ID: dedupe key value
     - Last Synced: now (ISO 8601)
   - Set Project relation: find Projects DB row by project_id, use its page_id.
   - Update Projects DB row: Current Phase = "Phase {phase_id}: {phase_name}", Sync Status = "OK".
   - Append to Activity: `"phase_started: Phase {phase_id}"`.
   - Field ownership guard: NEVER set Priority, Notes, Target Week, Area (on update — only on create).

2. `_sync_phase_completed(event: dict, result: SyncResult) -> None`:
   - Same dedupe key lookup as phase_started.
   - Update Status → "Done" (only if card is OpenClaw-linked — check openclaw_phase_id is non-empty).
   - Append to Activity: `"phase_completed: Phase {phase_id}"`.
   - Update Last Synced.
   - Update Projects DB: Current Phase cleared or updated to "Completed: Phase {phase_id}".

3. `_sync_phase_blocked(event: dict, result: SyncResult) -> None`:
   - Same dedupe key lookup.
   - Update Status → "Waiting" (only if OpenClaw-linked).
   - Append to Activity: `"phase_blocked: {blocker details from payload}"`.
   - Update Last Synced.

4. **Field ownership guard function** `_is_openclaw_linked(page: dict) -> bool`:
   - Returns True if page has non-empty `OpenClaw Phase ID` OR `OpenClaw Event Anchor` property.
   - Used before any Status write: if not linked, skip Status update (per SPEC status ownership carve-out).

5. **Status write guard** `_safe_set_status(properties: dict, page: dict, new_status: str) -> dict`:
   - If `_is_openclaw_linked(page)`: add Status to properties dict.
   - If not linked: skip Status (Notion owns it).
   - Returns modified properties dict.

6. **Project relation lookup** `_find_project_page_id(client: NotionClient, project_id: str) -> Optional[str]`:
   - Query Projects DB for OpenClaw ID = project_id.
   - Return page_id if found, None otherwise.
   - Cache result in module-level dict to avoid repeated queries.

All handlers pass the SyncResult instance and call `result.record_mutation()` with action, target, page_id, dedupe_key on success, or `result.record_error()` on failure.

CRITICAL: Every Notion API call is wrapped in try/except. On failure, record error in SyncResult and continue — never abort the entire sync for one failed mutation.
