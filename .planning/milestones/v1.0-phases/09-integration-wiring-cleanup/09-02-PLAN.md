Edit `buildAgentHierarchy()` in `workspace/occc/src/lib/metrics.ts` to add two validation checks BEFORE the existing map/return logic:

**1. reports_to referent validation:**
Build a Set of all agent IDs from the input array. For each agent with a non-null `reports_to`, check if the referenced ID exists in the set. If not, `console.warn` with a descriptive message:
```
[hierarchy] Agent '{agent.id}' reports_to '{agent.reports_to}' which is not in the agents list
```

**2. Circular chain detection:**
For each agent, walk the `reports_to` chain. If the chain length exceeds the total number of agents, a cycle exists. `console.warn`:
```
[hierarchy] Circular reports_to chain detected starting from '{agent.id}'
```

Implementation approach (from 09-RESEARCH.md Pattern 5):
```typescript
// Build ID set for referent validation
const agentIds = new Set(agents.map((a) => a.id));

// Validate reports_to references
agents.forEach((agent) => {
  if (agent.reports_to && !agentIds.has(agent.reports_to)) {
    console.warn(
      `[hierarchy] Agent '${agent.id}' reports_to '${agent.reports_to}' which is not in the agents list`
    );
  }
});

// Detect circular chains
agents.forEach((agent) => {
  let current = agent.reports_to;
  let steps = 0;
  while (current && steps <= agents.length) {
    const parent = agents.find((a) => a.id === current);
    current = parent?.reports_to ?? null;
    steps++;
  }
  if (steps > agents.length) {
    console.warn(`[hierarchy] Circular reports_to chain detected starting from '${agent.id}'`);
  }
});
```

CRITICAL: Use `console.warn` ONLY — never throw. This function runs in the dashboard render path. A throw would break the UI. The validation is advisory, not blocking.

Check the TypeScript types for the agent object (likely `OpenClawAgent` interface). If `level` and `reports_to` are not yet in the type definition, add them:
- `level?: number`
- `reports_to?: string | null`

These should be optional (`?`) so the type is backward-compatible with agents that lack the fields.
