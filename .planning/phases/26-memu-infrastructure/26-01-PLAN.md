```bash
#!/bin/bash
set -e
psql -v ON_ERROR_STOP=1 --username "$POSTGRES_USER" --dbname "$POSTGRES_DB" <<-EOSQL
    CREATE EXTENSION IF NOT EXISTS vector;
    ALTER SYSTEM SET maintenance_work_mem = '128MB';
    ALTER SYSTEM SET max_parallel_workers_per_gather = 4;
    SELECT pg_reload_conf();
EOSQL
```

Create `docker/memory/requirements.txt`:
```
fastapi>=0.115.0
uvicorn[standard]>=0.30.0
memu-py[postgres]>=1.4.0
```

CRITICAL: The PyPI package is `memu-py` (with hyphen), NOT `memu`. The `[postgres]` extra pulls pgvector, sqlalchemy[postgresql-psycopgbinary], and alembic.

Create `docker/memory/Dockerfile`:
```dockerfile
FROM python:3.13-slim-bookworm

WORKDIR /app

# Install build dependencies for psycopg binary
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY memory_service/ ./memory_service/

EXPOSE 18791

CMD ["uvicorn", "memory_service.main:app", "--host", "0.0.0.0", "--port", "18791", "--workers", "1"]
```

Base image: `python:3.13-slim-bookworm` per user decision. Single uvicorn worker is correct — memu-py is async internally and multiple workers would create separate MemUService instances with independent DB pools.
