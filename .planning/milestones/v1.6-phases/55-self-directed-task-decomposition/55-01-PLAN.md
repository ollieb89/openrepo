# Plan 55-01: Autonomy Runner Core & Events

## Objective
Implement the `l3_autonomy_runner.py` script that manages the self-directed task breakdown, LLM self-reflection validation, and sequential execution loop with event emission.

## Context
As defined in `55-CONTEXT.md`, the L3 agent must plan its execution before acting. The plan is generated at Claude's discretion, verified via an LLM self-reflection pass, and emitted to the Event Bus as a single `AutonomyPlanGenerated` event. Execution happens inline sequentially with detailed progress updates and heartbeats. Fallbacks involve auto-modifying remaining steps. 

## Requirements
- AUTO-01: L3 agents perform self-directed task breakdown and planning

<action>
Implement the `execution_phase` method in `runner.py`:
1. Loop over the generated steps sequentially.
2. For each step, invoke the CLI runtime to execute it. Start a background asyncio task for heartbeats if the step takes a long time.
3. Emit `AutonomyProgressUpdated` after each step finishes.
4. Catch errors: if a step fails, invoke the CLI runtime with the error to auto-modify the remaining steps (replanning), then retry.
5. If retries run out, transition the state to `BLOCKED` (via `hooks.on_task_failed`) and exit.
