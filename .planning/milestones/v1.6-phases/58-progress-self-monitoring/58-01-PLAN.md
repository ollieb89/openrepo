# Plan 58-01: Heuristics & Dynamic Reflection

## 1. Objectives
- Replace the simplistic hardcoded retry logic in `execution_phase` with a dynamic, LLM-driven course correction mechanism.
- Implement heuristic checks (`_detect_deviation`) to accurately identify when the agent is stuck or has failed.
- Safely splice recovery steps into the execution queue.
- Provide telemetry of course corrections via an `AutonomyCourseCorrection` event.

## 2. Context
Currently, the L3 runner implements a static retry loop (`retry_prompt`) when a step fails. This is often insufficient for complex deviations. We are shifting to a "Hybrid Approach" (AUTO-04) where failures, timeouts, and error keyword densities trigger a dedicated reflection step, producing fresh steps that are dynamically inserted into the remaining plan. 

## 3. Requirements
- The original failing step must be marked as failed and skipped ("Fail and Proceed").
- Confidence scores drop normally during failures but DO NOT reset during reflection—they only recover passively as the new recovery steps succeed ("Prove Recovery First").
- Emit an `AutonomyCourseCorrection` event to notify the Orchestrator without immediately demanding user intervention.

## 4. Tasks

1. **Update Events Module (`events.py`)**:
   - Define `AutonomyCourseCorrection` with `task_id`, `failed_step`, and `recovery_steps`.
   - Register it under `EVENT_COURSE_CORRECTION = "autonomy.course_correction"`.

2. **Add Deviation Heuristics (`runner.py`)**:
   - Add `_detect_deviation(self, success: bool, output: str, duration: float) -> bool`.
   - Return `True` if `success` is False.
   - Return `True` if `duration > 180` seconds.
   - Return `True` if combined count of "error", "exception", and "traceback" in `output.lower()` > 3.

3. **Add Reflection Logic (`runner.py`)**:
   - Add `_reflect_and_correct(self, failed_step: dict, output: str) -> list[dict]`.
   - Construct a prompt asking the LLM to diagnose the failure and provide exactly 1-2 new JSON steps to recover.
   - Parse the JSON using `_extract_json`. Return the `steps` array or `[]` on failure.

4. **Refactor Execution Phase (`runner.py`)**:
   - In `execution_phase`, remove the static `retry_prompt` fallback block.
   - Replace it with a block that checks `_detect_deviation()`. If true, call `_reflect_and_correct()`.
   - If recovery steps are returned, emit `AutonomyCourseCorrection` and splice the recovery steps into the `steps` array directly after the current index.
   - Ensure `self._evaluate_confidence` naturally handles the failure deductions so repeated failures eventually trigger the existing escalation hook.

5. **Update Unit Tests (`test_runner.py`)**:
   - Write `test_detect_deviation` testing the 3 heuristic branches.
   - Write `test_reflect_and_correct` testing JSON array extraction.
   - Refactor `test_execution_phase_fallback` into `test_execution_phase_course_correction` to verify dynamic step insertion.

## 5. Success Criteria
- The heuristics correctly flag errors and timeouts.
- The `execution_phase` loop cleanly expands when recovery steps are injected.
- Infinite course-correction loops are prevented by the `confidence_score` dropping below the escalation threshold.
