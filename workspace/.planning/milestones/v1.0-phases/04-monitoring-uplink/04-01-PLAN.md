Create `src/lib/jarvis.ts` with TypeScript types matching the Jarvis Protocol schema from `orchestration/state_engine.py`:
- `JarvisState` interface: `{ version: string, protocol: 'jarvis', tasks: Record<string, Task>, metadata: { created_at: number, last_updated: number } }`
- `Task` interface: `{ status: 'pending' | 'in_progress' | 'completed' | 'failed', skill_hint?: string, activity_log: ActivityEntry[], created_at: number, updated_at: number, metadata?: Record<string, any> }`
- `ActivityEntry` interface: `{ timestamp: number, status: string, entry: string }`
- Zod schemas for runtime validation of each type
- Export a `parseJarvisState(data: unknown): JarvisState` function that validates with zod and throws on invalid shape

Create `src/lib/metrics.ts` with:
- `AgentNode` interface: `{ id: string, name: string, level: 1 | 2 | 3, status: 'idle' | 'working' | 'error' | 'offline', currentTask?: string, reports_to?: string, container_name?: string }`
- `SwarmMetrics` interface: `{ totalByTier: { L1: number, L2: number, L3: number }, active: number, idle: number, errored: number, totalTasks: number, completedTasks: number, failedTasks: number }`
- `buildAgentHierarchy(agents: any[], state: JarvisState): AgentNode[]` function that maps openclaw.json agents to AgentNode entries by correlating agent IDs with state.json task metadata
- `deriveSwarmMetrics(agents: AgentNode[], state: JarvisState): SwarmMetrics` function that computes tier counts, status counts, and task throughput from current state
- Agent status derivation logic: if agent has in_progress tasks = 'working', if agent has failed tasks = 'error', if no tasks or all completed = 'idle', if container not found = 'offline'
  </action>

Rewrite `src/app/api/swarm/route.ts` to read real data instead of mock:
- Read state.json from `process.env.STATE_FILE` (default: path resolved relative to project -- for dev, use `../../.openclaw/workspace-state.json` or the absolute path `/home/ollie/.openclaw/workspace/.openclaw/workspace-state.json`)
- Read agents list from `process.env.OPENCLAW_CONFIG` (default: `../../openclaw.json` or absolute path)
- Parse state.json with `parseJarvisState()` from lib/jarvis.ts for validation
- Call `buildAgentHierarchy()` and `deriveSwarmMetrics()` from lib/metrics.ts
- Return JSON: `{ agents: AgentNode[], metrics: SwarmMetrics, state: JarvisState, lastUpdated: string }`
- On read error: return 500 with `{ error: 'Failed to read swarm state', detail: string }`
- Use file mtime check to cache parsed state in-memory with 500ms TTL (reduces file read contention per research pitfall 4)

Create `src/app/api/swarm/stream/route.ts` as SSE endpoint:
- Use Next.js App Router ReadableStream pattern for SSE
- Watch state.json for changes using `fs.stat()` mtime comparison at 1s interval
- When mtime changes, read new state and push SSE event: `data: {"updated": true}\n\n`
- Send keepalive comment every 30s: `: keepalive\n\n` (prevents proxy timeouts per research pitfall 3)
- Set headers: Content-Type: text/event-stream, Cache-Control: no-cache, Connection: keep-alive
- Clean up interval on client disconnect

Create `src/hooks/useSwarmState.ts` as the client-side hook:
- `"use client"` directive at top
- Use SWR with `/api/swarm` fetcher, `refreshInterval: 2000`, `revalidateOnFocus: true`, `revalidateOnReconnect: true`, `dedupingInterval: 2000`
- Layer SSE on top: create `EventSource('/api/swarm/stream')` in useEffect, call `mutate()` on message to force SWR revalidation
- Handle SSE errors with exponential backoff reconnection (1s, 2s, 4s, max 30s)
- Clean up EventSource on unmount
- Return: `{ agents: AgentNode[], metrics: SwarmMetrics, state: JarvisState, error: Error | undefined, isLoading: boolean }`
