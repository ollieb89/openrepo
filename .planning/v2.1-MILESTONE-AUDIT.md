## Integration Checker Additional Findings

### Critical Gap Details (Tech Debt, Non-Blocking)

**GAP-01: Live L1 Gateway Dispatch Blocked by `needs_ui_build`**
- `openclaw agent --agent clawdia_prime` fails: gateway shows `needs_ui_build` status, agent not registered
- `skills/router/index.js` code is correct (zero execFileSync, gateway-HTTP-only), but gateway runtime requires a UI build to register `clawdia_prime`
- Phase 79 substituted a Python socket dispatcher as functional equivalent for SSE pipeline testing — valid for INTG-01 verification but does not exercise the full L1→gateway HTTP→L2→L3 chain
- **Affects:** GATE-01, INTG-01 (operational gap, code is correct)

**GAP-02: AgentRegistry Not Consumed by Dashboard or Spawn Pipeline**
- `AgentRegistry.all_agents()` and `get_agent_registry()` are exported and the CLI works, but no dashboard API route, no `pool.py` spawn path, and no gateway dispatch path consumes `get_agent_registry()` at runtime
- Registry drift/orphan warnings emit to structured log output only — not surfaced in dashboard or automation
- `spawn.py` directly instantiates `AgentRegistry(project_root)` bypassing the `get_agent_registry()` factory
- **Affects:** AREG-01, AREG-02, AREG-03 (requirements verified by CLI and test; runtime integration is a future phase concern)

**GAP-03: Autonomy Alerts Silently Dropped Due to project_id='unknown'**
- `useAlerts.ts` → `useLiveEvents.ts` filters events by `event.project_id === projectId`
- `AutonomyEventBus` emits with `project_id="unknown"` — these events fail the filter and are silently dropped from the per-project alert feed
- Autonomy escalation alerts (`autonomy.escalation`, `autonomy.state_changed`) are never shown in the dashboard per-project alert panel
- **Affects:** EVNT-02 (partial — task-domain events work; autonomy-domain events silently filtered)

**GAP-04: `pool.max_concurrent` Hardcoded in Python Metrics Snapshot**
- `metrics.py:62` hardcodes `max_concurrent: 3` rather than reading per-project config
- Live snapshot always shows `3` regardless of project-specific `l3_overrides.max_concurrent`
- **Affects:** OBSV-01 (minor — data pipeline works; per-project accuracy is a deferred improvement)
