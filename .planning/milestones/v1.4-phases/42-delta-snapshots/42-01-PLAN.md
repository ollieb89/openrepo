**PERF-05 (JarvisState cursor helpers — 3 tests):**

**PERF-06 (spawn retrieval with cursor — 3 tests):**
- `test_retrieve_sends_created_after` — patches `httpx.Client` with a mock that captures the JSON payload; calls `_retrieve_memories_sync("http://fake", "proj", "query", created_after="2026-02-24T10:00:00+00:00")`; asserts the captured payload contains `"created_after": "2026-02-24T10:00:00+00:00"`. - `test_cursor_not_updated_on_fetch_failure` — calls `_retrieve_memories_sync` with a broken URL; checks that the second element of the returned tuple is `False` (indicating failure, so caller should not advance cursor)
- `test_cursor_updated_after_success` — patches `httpx.Client` to return a mock 200 response with `[]`; calls `_retrieve_memories_sync`; asserts second element of returned tuple is `True` (success, cursor should advance)
