# Plan 56-01: Confidence Math & Escalation Trigger

## 1. Objectives
- Pass confidence thresholds from the Orchestrator to the L3 container.
- Track a `confidence_score` in the L3 `AutonomyRunner`.
- Deduct confidence points based on step failures and output analysis.
- Emit `AutonomyConfidenceUpdated` and `AutonomyEscalationTriggered` events.

## 2. Context
Currently, `runner.py` retries once and immediately fails to `BLOCKED`. For Phase 56, we want a smoother confidence degradation. When confidence drops below a threshold, the state becomes `ESCALATING`.

## 3. Requirements
- The container must receive `AUTONOMY_CONFIDENCE_THRESHOLD` via environment variables.
- Confidence must start at 1.0.
- Repeated failures must deduct 0.3.
- Tool errors detected in LLM output (e.g., "command not found", "SyntaxError", "Error:") must deduct 0.15.
- Unclear LLM responses ("I need more context", "unclear") must deduct 0.5.
- If `confidence_score < threshold`, the runner must trigger escalation.

## 4. Tasks

1. **Modify `spawn.py`**:
   - Read `confidence_threshold` from `get_autonomy_config()`.
   - Inject `AUTONOMY_CONFIDENCE_THRESHOLD` into the container's environment dict.

2. **Modify `runner.py` Init & Config**:
   - Parse `AUTONOMY_CONFIDENCE_THRESHOLD` from `os.environ` (default: `0.4`).
   - Initialize `self.confidence_score = 1.0`.

3. **Implement Confidence Deduction in `runner.py`**:
   - Create `_evaluate_confidence(self, success: bool, output: str) -> None`.
   - Apply deductions based on `success` flag and string matching on `output`.
   - Emit `AutonomyConfidenceUpdated` when score changes.

4. **Trigger Escalation in `execution_phase`**:
   - Check `if self.confidence_score < self.confidence_threshold:`
   - Emit `AutonomyEscalationTriggered(reason="...", confidence=score)`.
   - Call `self._update_state("escalating", "Task escalated due to low confidence")`.
   - *Note: Pausing logic will be handled in 56-02. For now, it can still `sys.exit(1)` after escalating to keep tests simple before the next plan.*

5. **Write Unit Tests**:
   - Update `test_runner.py` to assert that failures deduct confidence properly.
   - Assert that `AutonomyEscalationTriggered` is fired when score drops low enough.

## 5. Success Criteria
- Confidence score accurately reflects task success/failure.
- Escalation is correctly triggered and emitted when the threshold is breached.
- Tests pass.
