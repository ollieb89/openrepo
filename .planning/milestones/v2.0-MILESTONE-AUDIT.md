### E2E Flow: Propose -> Approve -> Dashboard -> Confidence Chart

1. `openclaw-propose "outcome"` -> LLM pipeline -> 3 archetype proposals with rubric scores
2. User feedback -> soft correction via `CorrectionSession` -> re-proposal with diff display
3. User approves -> `approve_topology()` writes `current.json` + changelog with `rubric_scores` annotations (Phase 66 wiring)
4. `MemoryProfiler.compute_profile()` updates preference profile post-approval
5. L1 router `hasApprovedTopology()` gate passes -> L3 spawns now permitted
6. Dashboard `/topology` page loads via SWR hooks -> React Flow DAG, DualPanel, ProposalComparison
7. ConfidenceChart reads `annotations.rubric_scores` from changelog -> renders per-archetype confidence lines

**Full chain verified end-to-end.** Phase 61 models flow through Phase 62 proposer -> Phase 63 correction/approval -> Phase 64 memory -> Phase 65 dashboard -> Phase 66 rubric score wiring.
