**Makefile — `dashboard` target:**

Replace the current `dashboard` target:

```makefile
dashboard: ## Start dashboard dev server (port 6987)
	cd packages/dashboard && bun install && bun run dev
```

With a guarded version:

```makefile
dashboard: ## Start dashboard dev server (port 6987) — OPENCLAW_ROOT must be exported
	@if [ -z "$$OPENCLAW_ROOT" ]; then \
		echo "ERROR: OPENCLAW_ROOT is not set. The dashboard requires this to locate suggest.py and soul-suggestions.json."; \
		echo "  Run: export OPENCLAW_ROOT=$$HOME/.openclaw"; \
		exit 1; \
	fi
	cd packages/dashboard && bun install && bun run dev
```

Use `$$` (double dollar) for all shell variable references in the Makefile recipe — single `$` is expanded by Make and silently becomes empty. The guard form (exit 1) is correct per the research — soft-warn form allows silent failures that are the exact problem this item addresses.
