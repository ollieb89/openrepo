2. **Add SIGTERM trap to entrypoint.sh** — insert immediately after the `update_state()` helper function definition (after line 25, before the git config block):

```bash
# SIGTERM handler — graceful shutdown (REL-04, REL-05)
_shutdown_requested=0

_trap_sigterm() {
    [[ $_shutdown_requested -eq 1 ]] && return  # idempotent — ignore duplicate signals
    _shutdown_requested=1
    local elapsed=$(( $(date +%s) - _task_start_time ))
    echo "SIGTERM received after ${elapsed}s — writing interrupted state"
    # update_state uses JarvisState with LOCK_TIMEOUT=5; || true ensures exit even on lock failure
    update_state "interrupted" "SIGTERM received after ${elapsed}s. Container shutting down." || true
    # Kill child process (CLI runtime) if running
    [[ -n "${_child_pid:-}" ]] && kill "$_child_pid" 2>/dev/null || true
    exit 143  # 128 + 15 (SIGTERM)
}

trap '_trap_sigterm' TERM
_task_start_time=$(date +%s)
```

3. **Modify CLI runtime invocation** (around line 76) to run as a background process so the trap can fire while the runtime is running. Replace:
```bash
"${CLI_RUNTIME}" "${SOUL_ARGS[@]}" --task "${TASK_DESCRIPTION}" 2>&1 | tee /tmp/task-output.log || true
EXIT_CODE=${PIPESTATUS[0]}
```
With:
```bash
"${CLI_RUNTIME}" "${SOUL_ARGS[@]}" --task "${TASK_DESCRIPTION}" 2>&1 | tee /tmp/task-output.log &
_child_pid=$!
wait $_child_pid || true
EXIT_CODE=$?
```

This runs the CLI runtime in the background so bash (PID 1) remains in the foreground to receive SIGTERM. The `wait` blocks until the child finishes or is killed by the trap. Note: with pipes, `_child_pid` captures the last pipeline PID (tee), but killing tee causes the CLI runtime to get SIGPIPE which terminates it. This is acceptable behavior.

4. **Ensure the `set -e` does not interfere with the trap.** The `|| true` after `wait` prevents `set -e` from exiting on non-zero wait status. The trap handler itself uses `|| true` on both `update_state` and `kill` calls.

Do NOT modify the Dockerfile — it already uses exec form.
