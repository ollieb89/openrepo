  <action>
Replace the existing `_format_memory_context()` function in `skills/spawn_specialist/spawn.py` with a two-section variant.

**Changes to `_format_memory_context()`:**

1. Keep the same function signature: `def _format_memory_context(memories: list) -> str:`
2. Early return `""` if `not memories` (unchanged)
3. Create two accumulator lists: `work_bullets` and `review_bullets`
4. Maintain shared `total_chars` counter across both lists
5. For each item in memories:
   - Extract text from `item.get("resource_url", "") or item.get("content", "") or ""`
   - Skip empty text
   - Discriminate category: `is_review = (item.get("category", "") == "review_decision") or (item.get("agent_type", "") == "l2_pm")` — dual check as fallback per research recommendation (Open Question 1)
   - Build bullet: `f"- {text}"` (remove the old `{tag}` suffix — the section header now provides the source context)
   - Check budget: `total_chars + len(bullet) + 1 > MEMORY_CONTEXT_BUDGET` → break (whole-item drop, unchanged)
   - Append to `review_bullets` if is_review, else `work_bullets`
   - Increment `total_chars`
6. Build sections list:
   - If `work_bullets`: append `"## Past Work Context\n\n" + "\n".join(work_bullets)`
   - If `review_bullets`: append `"## Past Review Outcomes\n\n" + "\n".join(review_bullets)`
7. Return `"\n\n".join(sections) if sections else ""`

**Update the docstring** to document the two-section behavior, the shared budget, and the category discrimination logic.

**No changes to `_build_augmented_soul()`** — it receives the formatted string and appends it, so the section split is transparent.
