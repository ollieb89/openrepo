- Add a `set_task_metric(self, task_id: str, key: str, value: Any)` method that acquires LOCK_EX, reads state, sets `state['tasks'][task_id][key] = value`, and writes back via `_write_state_locked`.   1. Acquires LOCK_EX
  2. Reads the task's `activity_log` array
  3. If `len(activity_log) > ACTIVITY_LOG_MAX_ENTRIES`: archive the oldest entries by saving them to a new key `archived_activity_count` (integer, cumulative count of archived entries) and trim `activity_log` to the last `ACTIVITY_LOG_MAX_ENTRIES` entries
  4. Writes back via `_write_state_locked`
  5. Logs rotation at INFO level with task_id, entries_archived count, and remaining count
  6. If log is within threshold, no-op (no lock acquired, early return)
- Modify `update_task()` to call `self.rotate_activity_log(task_id)` after the successful write, outside the lock context (rotate_activity_log acquires its own lock). This ensures every activity append triggers a rotation check.
- Add a `_lock_wait_time(self, start_time: float) -> float` private helper that returns `(time.time() - start_time) * 1000` (milliseconds). Use this in `_acquire_lock` to record wait duration. Modify `_acquire_lock` to return the wait time in ms (change return type to `float` — 0.0 for immediate acquisition). Update all callers: `read_state`, `update_task`, `create_task`, `set_task_metric`, `rotate_activity_log` to capture the returned wait time. The cumulative lock wait per task is tracked by pool.py externally, not inside state_engine — state_engine just reports per-call wait times via return value.
