2. Update `src/lib/types.ts` — add `max_concurrent` to the `l3_overrides` interface so the metrics API can read pool limits:
```typescript
l3_overrides?: {
  mem_limit: string;
  cpu_quota: number;
  runtimes: string[];
  max_concurrent?: number;  // ADD: pool concurrency limit (default 3)
};
```
Also add a `MetricsResponse` interface:
```typescript
export interface MetricsResponse {
  completionDurations: { id: string; durationS: number }[];
  lifecycle: { pending: number; active: number; completed: number; failed: number };
  poolUtilization: number; // 0-100 percentage
  poolMax: number;
  poolActive: number;
  projectId: string;
}
```

3. Create `src/app/api/metrics/route.ts` — GET handler following the exact pattern in `/api/tasks/route.ts`:
- Accept `?project=` query param, fallback to `getActiveProjectId()`
- Call `getTaskState(projectId)` to get all tasks
- Call `getProject(projectId)` to get `l3_overrides.max_concurrent` (default 3)
- Compute `completionDurations`: filter tasks where `metadata.completed_at` AND `metadata.container_started_at` exist (both are numbers — epoch seconds stamped by pool.py's `set_task_metric()`), compute `durationS = Math.round((completed_at - container_started_at) * 10) / 10`, take last 15 sorted by `completed_at` ascending
- Compute `lifecycle`: count tasks by status — `pending` count, `active` = statuses `in_progress` + `starting` + `testing`, `completed` count, `failed` = `failed` + `rejected`
- Compute `poolUtilization`: `Math.round((activeCount / poolMax) * 100)`, clamped to 0-100
- Return `Response.json({ completionDurations, lifecycle, poolUtilization, poolMax, poolActive: activeCount, projectId })`
- Wrap in try/catch, return 500 on error (same pattern as tasks route)

IMPORTANT: Task metrics are in `metadata` — access as `(task.metadata.completed_at as number)` and `(task.metadata.container_started_at as number)`. These are set by `set_task_metric()` in pool.py.

4. Create `src/lib/hooks/useMetrics.ts` — SWR hook matching the `useTasks` pattern:
```typescript
import useSWR from 'swr';
import type { MetricsResponse } from '@/lib/types';

const fetcher = (url: string) => fetch(url).then(res => res.json());

export function useMetrics(projectId: string | null) {
  const { data, error, isLoading, mutate } = useSWR<MetricsResponse>(
    projectId ? `/api/metrics?project=${projectId}` : null,
    fetcher,
    { refreshInterval: 5000 }
  );
  return { metrics: data || null, isLoading, error, refresh: mutate };
}
```
