1. Add imports: `from typing import Literal, Optional` (extend existing `from typing import Any` import).
2. Define `CategoryValue = Literal["review_decision", "task_outcome"]` as a module-level type alias, placed ABOVE `MemorizeRequest`.
3. Add `category: Optional[CategoryValue] = None` field to `MemorizeRequest`, after the `user` field.

This gives strict Pydantic validation — unknown category values raise `ValidationError` (per locked decision: catches bugs early). `None` default is backward-compatible with existing callers.

In `docker/memory/memory_service/routers/memorize.py`:

1. In `_run_memorize()`, before the `await service.memorize(...)` call, build a new `user_dict`:
   - `user_dict = dict(request.user) if request.user else {}`
   - If `request.category is not None`, add: `user_dict["category"] = request.category`
   - This is a non-mutating merge (creates a new dict, does not modify `request.user` in place — per anti-pattern guidance from research).
2. Pass `user=user_dict if user_dict else None` to `service.memorize()` instead of `user=request.user`.

WHY user dict injection: memu-py's `MemorizeMixin.memorize()` signature is `(self, *, resource_url, modality, user)` — no `category` param. Adding `category=...` as a kwarg would raise `TypeError`. The user dict is the correct injection point because `merge_scope_model()` in memu-py uses `ConfigDict(extra="allow")`, so extra fields in the user dict are stored in the DB and returned on retrieval.
