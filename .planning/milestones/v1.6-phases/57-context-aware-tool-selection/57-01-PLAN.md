# Plan 57-01: Intent Analysis & Prompt Injection

## 1. Objectives
- Implement pre-planning context analysis to determine required tool categories.
- Store selected tool constraints in the L3 runner.
- Inject constraint warnings into all LLM prompts to soft-enforce tool boundaries.
- Emit a tracking event for selected tools.

## 2. Context
Currently, L3 agents have open access to all tools the underlying CLI (`claude-code`) natively supports (Bash, FileRead, FileEdit, etc.). We need to constrain these based on the task description to fulfill AUTO-03 (Context-Aware Tool Selection). We will achieve this via "soft constraints"â€”instructing the LLM directly through prompt injection which tools it is allowed to use.

## 3. Requirements
- Define `AutonomyToolsSelected` event.
- Analyze the `task_description` to select an array of tool categories (e.g., `["file_read", "file_write", "shell_execution"]`).
- Fallback to `["all"]` if the analysis fails or returns invalid JSON.
- Build a standardized constraint string that is appended to both the `planning_phase` prompt and the `execute_step` prompt.

## 4. Tasks

1. **Update Events Module (`events.py`)**:
   - Create `AutonomyToolsSelected` dataclass inheriting from `EventPayload`.
   - Fields: `task_id: str`, `selected_tools: List[str]`.
   - Register it under `EVENT_TOOLS_SELECTED = "autonomy.tools_selected"`.

2. **Add Analysis Method (`runner.py`)**:
   - In `AutonomyRunner`, add `_analyze_tool_requirements(self) -> List[str]`.
   - It should construct a prompt asking the LLM to choose from `[file_read, file_write, shell_execution, web_search, git_operations]` based on `self.task_description`.
   - The LLM must return ONLY a JSON array.
   - Parse the JSON array. On error or empty, return `["all"]`.
   - Emit `AutonomyToolsSelected` with the chosen tools.

3. **Prompt Injection (`runner.py`)**:
   - Add `_build_tool_constraint_prompt(self, tools: List[str]) -> str`.
   - If `"all"` in tools, return `""` (no constraints).
   - Otherwise, return `"\n\nCRITICAL INSTRUCTION: For this task, you are STRICTLY LIMITED to the following tool categories: " + ", ".join(tools) + ". Do not use any other tools."`.
   - In `run()` or at the start of `planning_phase`, calculate and store `self.active_tools = self._analyze_tool_requirements()`.
   - Append `self._build_tool_constraint_prompt(self.active_tools)` to the `prompt` inside `planning_phase()` and `execute_step()`.

4. **Update Unit Tests (`test_runner.py`)**:
   - Write tests for `_analyze_tool_requirements` (success parsing, and fallback on failure).
   - Write tests for `_build_tool_constraint_prompt`.
   - Update existing execution/planning tests since the runner will make an additional `_invoke_cli` call initially (meaning `mock_subprocess.side_effect` lists need an extra mock return value at the start for the tool selection step).

## 5. Success Criteria
- The runner successfully calls the LLM for tool analysis before planning.
- Constraints are correctly appended to the prompt strings.
- Unit tests verify the parsing and fallback logic.
- An event is emitted containing the tool profile.
