<task type="auto">
  <name>Task 1: Setup Suggestions Storage and Scoring Engine</name>
  <files>src/lib/sync/vector-store.ts, src/lib/sync/relevance.ts</files>
  <action>
    1. Update `src/lib/sync/vector-store.ts`: Add `link_suggestions` table.
       ```sql
       CREATE TABLE IF NOT EXISTS link_suggestions (
         id TEXT PRIMARY KEY,
         decision_id TEXT NOT NULL,
         issue_id TEXT NOT NULL,
         score REAL NOT NULL,
         status TEXT DEFAULT 'pending', -- 'pending', 'accepted', 'rejected'
         reasons JSON, -- e.g. ["explicit_mention", "semantic_similarity"]
         created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
         UNIQUE(decision_id, issue_id)
       );
       ```
    2. Create `src/lib/sync/relevance.ts`: Implement multi-signal scoring logic.
       - `calculateScore(decision, issue)`: Returns a score (0.0 to 1.0) and a list of reasons.
       - Signals:
         - **Explicit Mention:** Search for issue ID (e.g., "GH-123" or "LIN-45") in decision text. Boost score to 0.95 if found.
         - **Semantic Similarity:** Perform vector search using `sqlite-vss`. Score based on cosine similarity.
         - **Keyword Overlap:** Check for shared technical keywords (from a curated list or extracted from titles).
         - **Temporal Proximity:** Boost score if the decision and issue were created within a 48-hour window.
  </action>
  <verify>
    Verify `link_suggestions` table creation. Run unit tests for `calculateScore` with mock inputs representing the different signals.
  </verify>
  <done>
    Database is ready for suggestions and the scoring logic correctly weighs different signals.
  </done>
</task>
