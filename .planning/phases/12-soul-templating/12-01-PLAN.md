Create `orchestration/soul_renderer.py` with the following functions. Use `string.Template.safe_substitute()` for variable substitution (locked decision from STATE.md — not Jinja2). Import `_find_project_root` and `load_project_config` from `orchestration.project_config`.

**Functions to implement:**

1. `build_variables(project_config: dict) -> dict` — Maps project.json fields to template variable names. Must include:
   - `project_name` from `config["name"]` (fallback to `config["id"]`)
   - `project_id` from `config["id"]`
   - `agent_name` — derive from `config["agents"]["l2_pm"]` field, formatted for display. For `"pumplai_pm"`, this must produce `"PumplAI_PM"`. Implementation: use a lookup dict in project.json or the raw agent ID with custom formatting. Simplest approach: add an `"agent_display_name"` key to the variables dict sourced from `project_config.get("agent_display_name")`, falling back to `config["agents"]["l2_pm"]` title-cased. The PumplAI `project.json` needs `"agent_display_name": "PumplAI_PM"` added for golden baseline match.
   - `tier` — hardcoded `"L2"`
   - `tech_stack_frontend`, `tech_stack_backend`, `tech_stack_infra` from `config["tech_stack"]`
   - `workspace` — the OpenClaw runtime workspace path: `str(_find_project_root() / "workspace")`. NOT `project.json.workspace` (see RESEARCH.md Pitfall 1).

2. `parse_sections(text: str) -> tuple[dict, list]` — Parse markdown into `({section_name: section_text}, [ordered_names])`. Section text includes the `## ` header line and all content until the next `## ` header. Title line (`# ...`) is excluded. Use line-by-line iteration (not regex — see RESEARCH.md anti-patterns).

3. `merge_sections(default_sections, default_order, override_sections, override_order) -> str` — Merge override into default. Override replaces matching keys. Novel override sections appended at end. Reconstruct in default template order. Per locked decision: additive override model.

4. `render_soul(project_id: str) -> str` — Main entry point:
   a. Load project config via `load_project_config(project_id)`
   b. Build variables via `build_variables(config)`
   c. Load default template from `agents/_templates/soul-default.md`, apply `safe_substitute()`
   d. Parse default into sections
   e. If `projects/<id>/soul-override.md` exists, load it, apply `safe_substitute()`, parse into sections
   f. Merge sections
   g. Generate title line: `# Soul: $agent_name ($tier)` with `safe_substitute()`
   h. Return `title + "\n\n" + body` normalized with `.rstrip("\n") + "\n"` at the end (see RESEARCH.md Pitfall 2)

Also update `projects/pumplai/project.json` to add `"agent_display_name": "PumplAI_PM"` field for golden baseline agent name derivation.

Include module-level docstring explaining the module's purpose and the template + override mechanism.
  </action>
