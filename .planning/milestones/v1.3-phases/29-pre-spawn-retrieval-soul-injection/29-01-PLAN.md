**Function 1: `_retrieve_memories_sync(base_url, project_id, query)`**
- Sync retrieval from memU `/retrieve` using `httpx.Client` (NOT async MemoryClient — spawn runs in a thread via `run_in_executor`, sync httpx is safe and avoids event-loop conflicts).
- Returns `[]` on ANY exception (graceful degradation per RET-04).
- Guard: if `base_url` or `project_id` is falsy, return `[]` immediately.
- Payload format mirrors `memory_client.py:retrieve()`: `{"queries": [{"role": "user", "content": query}], "where": {"user_id": project_id}}`.
- Response handling: if list, return `data[:_RETRIEVE_LIMIT]`; if dict with "items" key, return `data["items"][:_RETRIEVE_LIMIT]`; else `[]`.
- Catch all exceptions, log at WARNING with `extra={"project_id": project_id, "error": str(exc)}`.

**Function 2: `_format_memory_context(memories)`**
- Converts list of memory dicts to a markdown `## Memory Context` section.
- Returns `""` (empty string) when `memories` is empty or all items exceed budget individually — no header, no placeholder (locked decision).
- Budget accounting: measure total rendered length of all bullets (including `"- "` prefix, content, source tag, `\n` separators) against `MEMORY_CONTEXT_BUDGET`.
- Trimming: drop whole items (lowest-ranked last) rather than truncating. Stop adding bullets when the next would exceed budget.
- Source tag logic: if `item.get("category") == "l2_review"` use `(from L2 review)`, otherwise `(from memory)`.
- Content extraction: `item.get("resource_url", "") or item.get("content", "") or ""`. Skip items with empty content.
- Final format: `"## Memory Context\n\n" + "\n".join(bullets)`.

**Function 3: `_build_augmented_soul(project_root, memory_context)`**
- Read L3 SOUL base from `project_root / "agents" / "l3_specialist" / "agent" / "SOUL.md"` directly (NOT `render_soul()` which generates L2 content — critical distinction from research).
- If file missing, return `""` (graceful).
- If `memory_context` is falsy, return base SOUL unchanged.
- Otherwise: `base.rstrip("\n") + "\n\n" + memory_context + "\n"`.

**Function 4: `_write_soul_tempfile(content)`**
- Uses `tempfile.NamedTemporaryFile(mode="w", suffix=".soul.md", prefix="openclaw-", delete=False)`.
- Writes content, flushes, closes.
- Returns `Path(tmp.name)` for caller to use in volumes and cleanup.

**Retrieval block:**
```python
# --- Pre-spawn memory retrieval and SOUL injection ---
memu_cfg = get_memu_config()
memu_url = memu_cfg.get("memu_api_url", "")
query = f"{task_description} skill:{skill_hint}"
memories = _retrieve_memories_sync(memu_url, project_id, query)
memory_context = _format_memory_context(memories)

if memory_context:
    bullet_count = sum(1 for line in memory_context.splitlines() if line.startswith("-"))
    logger.info(
        f"Injected {bullet_count} memories ({len(memory_context)} chars) into SOUL",
        extra={"task_id": task_id, "project_id": project_id},
    )

soul_content = _build_augmented_soul(project_root, memory_context)
soul_tempfile = None
if soul_content:
    soul_tempfile = _write_soul_tempfile(soul_content)
    container_config["volumes"][str(soul_tempfile)] = {
        "bind": SOUL_CONTAINER_PATH,
        "mode": "ro",
    }
    container_config["environment"]["SOUL_FILE"] = SOUL_CONTAINER_PATH
```

**Modify the spawn + return block** to wrap `client.containers.run()` in try/finally for tempfile cleanup:
```python
try:
    container = client.containers.run(**container_config)
finally:
    if soul_tempfile:
        soul_tempfile.unlink(missing_ok=True)

return container
```

The tempfile is safe to delete after `containers.run()` returns because Docker bind-mounts by inode — the container retains read access after the host file is unlinked.

**Logging:**

- When retrieval returns empty: no log (silent — normal case for new projects).
- When retrieval fails: WARNING log from `_retrieve_memories_sync` (already handled in Task 1).
