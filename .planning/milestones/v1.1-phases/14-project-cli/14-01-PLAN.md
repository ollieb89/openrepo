<task type="auto">
  <name>Task 2: Create project_cli.py with all subcommands</name>
  <files>orchestration/project_cli.py</files>
  <action>
Create `orchestration/project_cli.py` following the established `monitor.py` argparse pattern. Module must be runnable as `python3 orchestration/project_cli.py <init|list|switch|remove>`.

**Imports and helpers:**
- Import `_find_project_root`, `get_active_project_id` from `orchestration.project_config` (with try/except ImportError fallback for script mode, same pattern as init.py)
- Import `write_soul` from `orchestration.soul_renderer` (lazy import inside init handler to avoid circular import — per Phase 18 pattern)
- Copy `Colors` class from `init.py` (ANSI codes: GREEN, RED, YELLOW, BLUE, RESET, BOLD)
- `_is_interactive()`: returns `sys.stdin.isatty()`
- `_validate_project_id(project_id)`: regex `^[a-zA-Z0-9-]{1,20}$` — same pattern as spawn.py. Raises ValueError on invalid.
- `_list_projects(root)`: enumerate `projects/` directory, skip `_`-prefixed entries. For each dir, try loading `project.json`. On failure, mark as `corrupt: True`. Return list of dicts.
- `_has_running_l3_containers(project_id)`: lazy-import `docker` inside function body (per Research anti-pattern guidance). Use `client.containers.list(filters={"label": f"openclaw.project={project_id}"})`. Return True if any containers found. On Docker import/connection failure, return False (err on side of allowing switch) and print a yellow warning.
- `_set_active_project(project_id, root)`: read-modify-write `openclaw.json` with `indent=2` and trailing newline.

**`init` subcommand (CLI-01, CLI-05):**
- Flags: `--id`, `--name`, `--template` (choices: fullstack, backend, ml-pipeline), `--workspace`, `--force`
- If `--id` or `--name` missing: prompt interactively if `_is_interactive()`, else error and exit(1)
- Validate project ID with `_validate_project_id()`
- ID collision check: if `projects/<id>/project.json` exists and `--force` not set, prompt "Project X exists. Overwrite? [y/N]" in interactive mode, error+exit in non-interactive
- Build default project.json dict: `id`, `name`, `agent_display_name` (`f"{name.replace(' ', '')}_PM"`), `workspace` (default: `<root>/workspace/<id>/`), `tech_stack` (empty strings), `agents` (`l2_pm: <id>_pm`, `l3_executor: l3_specialist`), `l3_overrides` (4g mem, 100000 cpu, 3 runtimes)
- If `--template`: load template from `projects/_templates/<template>.json`, merge tech_stack and l3_overrides into the default dict (template values are base, explicit flags override)
- If `--workspace`: override the workspace path
- Write `projects/<id>/project.json` with `mkdir(parents=True, exist_ok=True)` on parent dir
- Call `write_soul(project_id, skip_if_exists=not force_flag)` — lazy import from `orchestration.soul_renderer`. Print the output path.
- Auto-activate: call `_set_active_project(project_id, root)`
- L2 agent validation warning: after writing, check if `config["agents"]["l2_pm"]` exists in `openclaw.json:agents.list[*].id`. Print yellow warning if not found (non-fatal, per Research open question 1 and STATE.md blocker).
- Print green success: "Created project '<id>' (now active)"

**`list` subcommand (CLI-02):**
- Call `_list_projects(root)` to enumerate all projects
- Call `get_active_project_id()` to determine which is active (catch exceptions — no active project is valid state)
- Print formatted table with columns: ID (15 wide), NAME (15 wide), WORKSPACE (40 wide, truncated with ...), ACTIVE (* marker)
- Use Colors for header styling (BOLD)
- Corrupt projects show `(corrupt)` in NAME column, `--` in WORKSPACE column
- If no projects found, print "No projects found."

**`switch` subcommand (CLI-03):**
- Positional arg: `project_id`
- Validate project exists: check `projects/<id>/project.json` exists. If not, print error and exit(1).
- Check corrupt: try loading project.json. If invalid JSON, print error and exit(1).
- Guard: call `_has_running_l3_containers(get_active_project_id())` — if True, print red error "Cannot switch: L3 containers are running for project '<current>'. Wait for tasks to complete or stop containers." and exit(1). Note: we check the CURRENT active project, not the target.
- Call `_set_active_project(project_id, root)`
- Print green success: "Switched to project '<id>'"

**`remove` subcommand (CLI-04):**
- Positional arg: `project_id`
- Guard: if `project_id == get_active_project_id()`, print red error "Cannot remove the active project. Switch to another project first." and exit(1)
- Check project exists: if `projects/<id>/` doesn't exist, print error and exit(1)
- Confirmation: unless `--force`, prompt "Remove project '<id>' and all its files? [y/N]". In non-interactive mode without --force, error and exit(1)
- Delete `projects/<id>/` directory with `shutil.rmtree()`. Do NOT delete `workspace/<id>/` (locked decision: workspace is preserved).
- Print green success: "Removed project '<id>' (workspace preserved at workspace/<id>/)"

**`main()` function:**
- Create parser with `description='OpenClaw Project Manager'`
- Add four subparsers (init, list, switch, remove)
- Parse args, dispatch to handler functions
- `if __name__ == '__main__': sys.exit(main())`

**Error message convention:** Use `sys.stderr` for errors, `sys.stdout` for normal output. Exit code 0 for success, 1 for errors.

**Update the error message in project_config.py:** Change line 70 from the old "Create it with: mkdir..." message to reference the new CLI: `"Create it with: python3 orchestration/project_cli.py init --id {project_id}"`. This is a single-line edit in `orchestration/project_config.py`.
